Implemented Phase 50: Evidence Package Export — the single artifact that converts Intellios from a process tool to a regulatory defense asset. - Filename: `evidence-package-[name]-v[version]-[date].json`

## Decisions

- **Broader role access than compliance export**: Evidence package accessible to `designer | reviewer | compliance_officer | admin` — the designer needs to show this to their CRO/CISO, and restricting it to compliance_officer would block the primary buyer-facing use case
- **Reuse `assembleMRMReport()`**: The MRM report assembler already has all 14 sections including everything the strategic plan specified. No new data model or new assembly logic was needed — pure UX gap
- **No PDF generation this phase**: PDF generation requires a headless render pipeline. The existing MRM report page at `/blueprints/[id]/report` is print-optimized (CSS `print:` classes, page-break-before directives). The "View Compliance Report" link provides the print-to-PDF path. The JSON bundle is the machine-readable artifact for examiner tooling
- **Status gate preserved**: Requires `approved | deployed` — only complete evidence chains are exportable. In-progress work is not evidence
