# Session 057 — 2026-03-16

## Summary

Debugged and fixed the Stakeholder AI Agent chat component that was introduced in session 055-056.

## Actions

### 1. Root cause: stale Turbopack chunk
After rewriting `stakeholder-ai-chat.tsx` in the prior session, the browser continued serving the old compiled chunk (`f6bad7f2`) which contained the pre-rewrite code with `useChat` v4/v5 API. `window.location.reload(true)` did not clear it. Fix: stopped preview server, deleted `.next` cache entirely (`rm -rf src/.next`), restarted — forced a full recompile.

### 2. Root cause: wrong tool schema format
Server was returning `400` from Anthropic API: `tools.0.custom.input_schema.type: Field required`. The stakeholder-chat route used `parameters: z.object(...)` (old AI SDK v4/v5 style) instead of `inputSchema: zodSchema(...)` (AI SDK v6 style used everywhere else in the codebase). Fixed by:
- Adding `zodSchema` import from `"ai"`
- Changing `parameters: z.object({...})` → `inputSchema: zodSchema(z.object({...}))`
- Correcting model ID from `"claude-haiku-4-5"` → `"claude-haiku-4-5-20251001"`

### 3. Root cause: unstable useChat chat ID
Even after the schema fix, AI responses were not rendering in the UI despite valid 200 streaming responses. Root cause: React strict mode double-invokes `useEffect`. On first mount, `sendMessage` was called; on remount, `initSentRef.current` prevented a second call. But React's strict mode created a new `useChat` instance with a different random chat ID on remount — the streaming response was associated with the old ID and never appeared in the new instance's state. Fix: pass `id: \`stakeholder-${sessionId}-${domain}\`` to `useChat` for a stable, deterministic chat session key.

### Result
End-to-end flow verified:
- AI Interview opens with domain selector
- Selecting "Compliance" auto-sends initial greeting
- Claude Haiku responds with domain-specific first question (~800ms)
- Conversation thread renders correctly in the chat panel
- Input field enabled for stakeholder replies
