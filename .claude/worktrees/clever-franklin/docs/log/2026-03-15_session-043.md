# Session 043 — Phase 38: Classification-First Adaptive Intake

**Date**: 2026-03-15
**Branch**: feature/phases-13-36-vigil
**Phase**: 38

---

## Summary

Implemented Phase 38 — Classification-First Adaptive Intake. The intake system is now classification-first: immediately after Phase 1 context submission, the agent is classified into a type and risk tier. All downstream behavior — conversation depth, required stakeholder domains, finalization gating, and blueprint generation quality — is driven from this classification.

---

## Actions

### Item 1 — Foundation: Types + classify.ts + migration + schema

- **`src/lib/types/intake.ts`**: Added `AgentType`, `IntakeRiskTier`, `IntakeClassification` types with JSDoc
- **`src/lib/regulatory/classifier.ts`**: Added `IntakeRiskTier` import and exported `deriveRiskTierFromContext(ctx)` wrapper around private `euAiActRiskTier()` — maps EU AI Act tier to IntakeRiskTier deterministically
- **`src/lib/intake/classify.ts`** (NEW): Two-step classification service — `deriveRiskTierFromContext` for riskTier (no LLM), single Haiku `generateObject` call for agentType; safe default on error
- **`src/lib/db/migrations/0019_intake_classification.sql`** (NEW): `ALTER TABLE intake_sessions ADD COLUMN IF NOT EXISTS agent_type TEXT, risk_tier TEXT`
- **`src/lib/db/schema.ts`**: Added `agentType` and `riskTier` columns to `intakeSessions` table

### Item 2 — Context route trigger + classification override route

- **`src/app/api/intake/sessions/[id]/context/route.ts`**: Added fire-and-forget `classifyIntake()` call after context save; writes results to DB async without blocking response
- **`src/app/api/intake/sessions/[id]/classification/route.ts`** (NEW): `PATCH` endpoint — lets designers correct misclassification; validates session + enterprise access; updates agentType and/or riskTier

### Item 3 — Adaptive system prompt + chat route

- **`src/lib/intake/system-prompt.ts`**: Added 5th optional `classification?` parameter to `buildIntakeSystemPrompt`; added `buildClassificationBlock()` that injects agent type, risk tier, and tier-adaptive depth instructions (low=streamlined 5-8 turns, medium=current, high=deep probe, critical=exhaustive)
- **`src/app/api/intake/sessions/[id]/chat/route.ts`**: Loads `agentType`/`riskTier` from session, constructs `IntakeClassification`, passes as 5th arg to `buildIntakeSystemPrompt`; passes `getRiskTier` closure to `createIntakeTools`

### Item 4 — UI classification display + correction

- **`src/app/intake/[sessionId]/page.tsx`**: Added classification state; polls session endpoint at 1.5s intervals (max 10 polls) after context submit until `agentType` non-null; classification header shows type pill (gray) + risk tier badge (color-coded) + "Edit" button; edit mode shows dropdowns → PATCH → updates state; skeleton animation while loading

### Item 5 — Tier-aware stakeholder domain gating

- **`src/lib/intake/coverage.ts`**: Added `riskTier` optional param to `getExpectedContributionDomains` and `getMissingContributionDomains`; low=return [], high=add compliance+security, critical=add compliance+security+legal+risk
- **`src/lib/intake/tools.ts`**: Added `riskTier` param to `checkGovernanceSufficiency`; low=skip most checks (only capture verification gate); high=require compliance+audit; critical=require all 5 policy types; added `getRiskTier` optional param to `createIntakeTools`
- **`src/components/intake/stakeholder-contributions-panel.tsx`**: Added `riskTier` prop, threads to `getMissingContributionDomains`
- **`src/components/intake/intake-progress.tsx`**: Added `riskTier` prop, threads to `StakeholderContributionsPanel`
- **`src/components/intake/intake-review.tsx`**: Added `riskTier` prop, threads to both `getMissingContributionDomains` call sites

### Item 6 — Generation engine context + classification

- **`src/lib/generation/system-prompt.ts`**: Added `context` and `classification` optional params to `buildGenerationSystemPrompt`; new `buildContextClassificationBlock()` appends agent context, data sensitivity → `ownership.dataClassification` mapping, and tier-adaptive governance depth instruction
- **`src/lib/generation/generate.ts`**: Updated `generateBlueprint` signature to accept `intakeContext` and `classification` parameters; passes both to `buildGenerationSystemPrompt`
- **`src/app/api/blueprints/route.ts`**: Reads `intakeContext`, `agentType`, `riskTier` from session; constructs `IntakeClassification`; passes both to `generateBlueprint`
