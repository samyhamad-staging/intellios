## Actions

### 1. Admin Overview redesign (`src/app/page.tsx`)

**Problem:** The admin view had 10 cards above the agent list — 6 pipeline status cards (including zero-count terminal states) plus 4 redundant quick-action cards (Pipeline Board, Deploy, Review Queue, Dashboard) that duplicated the sidebar.

**Changes:**
- Removed 4 quick-action cards entirely
- Made the 4 active pipeline stage cards (Draft, In Review, Approved, Deployed) clickable `<Link>` wrappers routing to their respective pages
- Added `actionCallouts` array — amber callout when blueprints are in review, green callout when agents are approved for deploy — only renders when count > 0
- Demoted Rejected/Deprecated to a compact `0 Rejected · 1 Deprecated` text line below the cards
- Added `NewIntakeButton` to the header
- Removed unused `Rocket`, `BarChart3` imports

### 2. Dashboard cleanup (`src/app/dashboard/page.tsx`)

- Removed "Status" column from Recent Deployments table (always read "Deployed" — redundant)

### 3. Deployment Console cleanup (`src/app/deploy/page.tsx`)

**Changes:**

- Removed unused `CheckCircle` and `Globe` imports

### 4. Git connection + production deployment

Vercel was previously deployed via `vercel deploy` CLI from within `src/`. When connecting Git, Vercel cloned the repo from the root where there is no `package.json` — first Git-triggered build failed with `Module not found: Can't resolve '../node_modules/geist/...'` (wrong root directory, not actually a CSS path issue).

**Root cause:** Vercel Settings → Build and Deployment → Root Directory was set to `./` (repo root). The `package.json`, `next.config.ts`, and `node_modules` are all inside `src/`.

**Fix:**
1. Vercel Settings → Build and Deployment → Root Directory: `./` → `src`
2. Pushed `git commit --allow-empty` to trigger rebuild
3. Build succeeded in 1m 8s — deployment `7Duhy5Z8f` marked Current

**Production URL:** https://intellios.vercel.app — confirmed showing updated UI.
