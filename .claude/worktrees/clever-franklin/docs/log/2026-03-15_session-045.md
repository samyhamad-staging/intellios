## Summary

Applied design system and UX/performance optimizations identified in the prior session's analysis. Four targeted improvements: button color standardization, intake loading spinner, pipeline board mobile overflow fix, and an aggregated dashboard summary API.

## Actions

1. **Standardized primary button color to violet-600 across all pages.**
   - Replaced `bg-gray-900` with `bg-violet-600 hover:bg-violet-700` in:
     - `src/components/chat/chat-input.tsx` (Send button)
     - `src/components/intake/new-intake-button.tsx` (Design a New Agent)
     - `src/app/login/page.tsx` (Sign in)
     - `src/app/blueprints/[id]/page.tsx` (Run Tests, Save Ownership, Apply Changes)
   - Replaced `bg-indigo-600` → `bg-violet-600` in `lifecycle-controls.tsx` (Deploy to Production)
   - Replaced `bg-blue-600` → `bg-violet-600` in `src/app/admin/users/page.tsx` (Create User)
   - Updated focus ring: `focus:ring-gray-900` → `focus:ring-violet-500` in blueprint workbench textarea
   - Kept semantic colors: green (Approve), red (Reject), gray-400 (Deprecate), yellow (Submit for Review)

2. **Added animated spinner to intake session loading state.**
   - `src/app/intake/[sessionId]/page.tsx` — replaced plain text "Loading session…" with a violet SVG spinner + label

3. **Fixed pipeline board mobile overflow.**
   - `src/app/pipeline/page.tsx` — added `overflow-hidden` to the outer flex container (creates proper scroll boundary for the inner `overflow-x-auto` board)
   - Changed header controls from `flex items-center gap-3` → `flex flex-wrap items-center gap-2` so search/filter controls wrap on narrow viewports

4. **Created `/api/dashboard/summary` aggregated endpoint.**
   - `src/app/api/dashboard/summary/route.ts` (NEW)
   - Single DB query fetches agents with minimal fields (no blueprint JSON blob); separate COUNT query for active policies
   - Returns pre-computed: pipeline counts (7 statuses), governance health (clean/withErrors/notValidated/complianceRate), policyCount, top-5 recentDeployed, top-5 needingAttention
   - Updated `src/app/dashboard/page.tsx` to use the new endpoint instead of parallel `fetch("/api/registry")` + `fetch("/api/governance/policies")`; removed all client-side count computation; removed unused `TrendingUp`/`TrendingDown` imports

5. **TypeScript check:** `tsc --noEmit` → 0 errors in production code.
