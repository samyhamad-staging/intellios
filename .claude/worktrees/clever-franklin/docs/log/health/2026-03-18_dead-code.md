### HD-20260318-001: REPLACE_ME ARN placeholder as default parameter value
File: `src/lib/agentcore/translate.ts:124`

```typescript
agentResourceRoleArn = "arn:aws:iam::ACCOUNT_ID:role/AmazonBedrockExecutionRoleForAgents_REPLACE_ME",
```

The function `translateAbpToBedrockAgent` uses this placeholder as a default value for the optional `agentResourceRoleArn` field in `TranslateOptions`. This means if a caller omits the ARN, the generated Bedrock definition will silently contain an invalid ARN instead of failing fast.

Decision needed: Should this default be removed (making `agentResourceRoleArn` required), or replaced with an env var lookup (`process.env.BEDROCK_EXECUTION_ROLE_ARN`)? The current approach could cause a hard-to-diagnose Bedrock deployment failure downstream.
