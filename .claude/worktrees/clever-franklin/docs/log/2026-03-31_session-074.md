### Architecture Decision: `messageMetadata`

Chose AI SDK v6's `messageMetadata` callback on `toUIMessageStreamResponse()` over `DataUIPart` custom stream channels:
- No new API endpoints — metadata flows through existing streaming response
- No polling — data arrives with each response completion
- No transport changes — `DefaultChatTransport` consumes natively
- Zero latency impact — metadata computed after streaming, not blocking

## Decisions

- **`messageMetadata` is the right delivery mechanism for per-response transparency data.** No new endpoints, no polling, no transport changes. If mid-stream updates are needed later (Phase C decision trace), `DataUIPart` can be added alongside without conflicting.

- **Transparency panels are collapsible by default.** Progressive disclosure keeps the sidebar clean for users who don't need visibility, while making every AI decision one click away for users who do.

- **Tool call result status is derived from AI SDK v6's `state` field.** No custom tracking needed — the SDK already provides `output-available`, `output-error`, and `input-streaming` states per tool invocation.
