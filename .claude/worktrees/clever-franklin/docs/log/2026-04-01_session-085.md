# Session 085 — 2026-04-01

## Summary

Two rounds of UX improvements to the Agent Design Studio (intake session page). Round 1 addressed experience gaps identified after the session 084 redesign (zero-progress on revisit, unconstrained chat width, non-interactive domain strip, panel label renames, page header rename, sidebar label updates). Round 2 delivered six targeted polish items eliminating anxiety signals and navigation confusion.

## Actions

### Context

- Continued from session 084 (AI/futuristic redesign). This session applies polish and interaction fixes on top of that foundation.
- No new files created. Five existing files modified.

### Round 1 — Experience redesign completions

1. **Zero-progress on revisit fixed** — Added payload fetch + `useMemo` for `computedDomains` in `src/app/intake/[sessionId]/page.tsx`. Domain fill levels now restore correctly when returning to an in-progress session.
2. **Chat width constrained** — Applied `max-w-4xl mx-auto` to the message area in `src/components/chat/chat-container.tsx`. Chat no longer stretches to full viewport width on large screens.
3. **Domain strip made interactive** — Added `onDomainClick` prop, clickable chips, hover styles, and tooltip to `src/components/intake/domain-progress-strip.tsx`. Clicking a chip navigates to that domain in the conversation.
4. **Right panel labels renamed** — In `src/components/intake/intake-progress.tsx`: "SYSTEM ANALYSIS" → "DESIGN INTELLIGENCE", "AWAITING SIGNAL" → "ANALYZING", "Probing Topics" → "Coverage Analysis".
5. **Page header renamed** — "Agent Intake" → "Agent Design Studio" throughout.
6. **Sidebar labels updated** — In `src/components/nav/sidebar.tsx`: Design Studio, Analytics, Audit Trail, Policies, Team.

### Round 2 — UX polish

1. **Progressive disclosure** in Required Governance and Coverage Analysis sections — satisfied/covered items shown first; pending/open items capped at 3 with "+N more" expand button. Reduces visual clutter when most items are already satisfied. (`intake-progress.tsx`)
2. **Removed red dots from domain chips** — Dots were redundant with fill bars and caused anxiety when 5 appeared simultaneously. (`domain-progress-strip.tsx`)
3. **Replaced score ring with monospace counter** — SVG arc ring read as a "failing grade" percentage; `X/7 domains` counter reads as progress toward completion. (`domain-progress-strip.tsx`)
4. **Collapsed Stakeholder Input during ANALYZING state** — Previously the full stakeholder list contradicted the analyzing state. Now collapses to a single muted row when state is ANALYZING. (`intake-progress.tsx`)
5. **Differentiated navigation messages** — Domain chip clicks produce a ghost pill style (bg-primary/8, border, italic, Navigation icon) instead of a full dark user bubble. Navigation intent tracked via `useRef<Set<string>>`. (`chat-container.tsx`, `message-bubble.tsx`)
6. **Fixed active domain stuck on Behavior** — Added `pendingActiveDomain` optimistic override state; chip click immediately highlights the clicked domain without waiting for AI response metadata. (`src/app/intake/[sessionId]/page.tsx`)
