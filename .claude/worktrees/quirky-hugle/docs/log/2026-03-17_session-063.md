## Viewer Role Access Summary

**Can access (read-only):**
- Overview / Fleet Governance Dashboard
- Pipeline Board (read)
- Agent Registry (read)
- Governance Hub (read — no policy management)
- Compliance posture
- Monitor (read — no health check trigger)
- Fleet analytics dashboard
- Audit trail (enterprise-scoped)
- MRM reports
- Evidence packages

**Cannot do:**
- Create/edit blueprints (no intake, no studio actions)
- Approve/reject/deploy agents
- Create/edit/delete governance policies
- Trigger health checks or periodic reviews
- Manage users or enterprise settings
- Export code (no approved blueprints to export anyway)

## Decisions

- Viewer nav: intentionally excludes Review Queue (viewer doesn't approve), Deploy (viewer doesn't deploy), and all admin routes.
- `canCheck` in monitor page remains `compliance_officer | admin` — triggering live AWS health checks is a side-effecting action.
- `canManagePolicies` in governance page remains `compliance_officer | admin` — policy management is not read-only.
- Code export gate: uses `ErrorCode.BAD_REQUEST` (400), consistent with the 422 used by agentcore/compliance exports but intentionally 400 since this is a malformed request given the state constraint.
