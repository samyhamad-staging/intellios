## Summary

**MRM Compliance Report** — a single JSON artifact downloadable on demand from any Blueprint
detail page (compliance_officer and admin only). Covers 10 sections:

1. **Cover** — agent name, status, version, enterprise scope
2. **Risk Classification** — derived risk tier (High/Medium/Low from policy types), intended use,
   business owner, model owner, with explicit derivation notes for human validation
3. **Agent Identity** — name, description, persona, tags
4. **Capabilities** — tool inventory (count + type), knowledge sources, instructions status
5. **Governance Validation** — valid/invalid, policy count, full violation table with suggestions
6. **Review Decision** — outcome, reviewer, timestamp, verbatim rationale comment
7. **SOD Evidence** — designer/reviewer/deployer identities; `sodSatisfied` flag (all distinct)
8. **Deployment Change Record** — changeRef, deployment notes, deployer, deployment timestamp
9. **Model Lineage** — full version history (all agent versions) + deployment lineage
   (every production deployment event across all versions, with changeRef)
10. **Audit Chain** — all immutable lifecycle events for this blueprint version

Every export is itself written to the audit log (`blueprint.report_exported`), so regulators
can ask when a report was pulled and by whom.
