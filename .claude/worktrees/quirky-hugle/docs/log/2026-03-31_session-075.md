## Actions

### P1-001 — Mobile sidebar layout
**Problem:** `IntakeProgress` sidebar uses `w-72` (288px) with no responsive breakpoints. On a 375px mobile viewport the sidebar leaves only ~87px for the chat — completely unusable.

**Fix:**
- `src/components/intake/intake-progress.tsx`: Both `<aside>` elements changed from `flex flex-col` to `hidden lg:flex flex-col`. Added `mobileOpen` prop to `IntakeProgressProps`. When `mobileOpen` is true, `aside` uses `flex absolute inset-0 z-10 lg:relative lg:inset-auto` — full-viewport overlay on narrow screens, normal sidebar on large screens.
- `src/app/intake/[sessionId]/page.tsx`: Added `mobileSidebarOpen` state (default false). Added `lg:hidden` "Progress" toggle button in conversation-phase header. Added `mobileOpen={mobileSidebarOpen}` prop to `<IntakeProgress>`. Made `<main>` `relative` for overlay positioning.

### P1-002 — Classification loading resilience
**Problem:** When `submit_intake_context` saves context but classification fails (API timeout, network blip), `page.tsx` sets `classificationLoading: true` with no ceiling. The spinner runs indefinitely until the next `refreshTick` — and if classification never arrives, it runs forever.

**Fix:** Added `classificationLoadingTicksRef` (useRef) in `page.tsx`. In the `checkStatus` useEffect: when context is present but `agentType` is absent, increment the ref. If `≤ 2` ticks, show spinner. After 2 unanswered ticks, clear the spinner and let the UI fall back to a blank classification row. When classification eventually arrives, reset the counter to 0.

### P1-003 — onContextSubmit cold-path robustness
**Problem:** In `route.ts`, the context save (`db.update`) was unwrapped — if it threw, the error propagated as an unhandled exception through the tool's `execute()`, surfacing as a confusing stream error rather than a clean tool failure. Claude had no structured way to relay this to the user.

**Fix:** Wrapped the initial `db.update` in a try/catch. On failure: `console.error` with requestId context, then `throw new Error("Failed to save context — please try again")`. The tool's execute will surface this to Claude as a tool error, which Claude can relay as a clear user-facing message. Classification failure path unchanged — it remains non-fatal, returning null.
