### Step 2 — Agent Clone API

**Created:** `src/app/api/blueprints/[id]/clone/route.ts`

- `POST /api/blueprints/[id]/clone` with optional `{ name?: string }` body
- Access: `designer | admin` only
- Creates new `agentId` + new blueprint `id` (both `randomUUID()`) — clone is a new logical agent, not a new version
- Resets lifecycle: `status: "draft"`, `version: "1.0.0"`, `refinementCount: "0"`, all review fields cleared
- Updates `abp.identity.name` to clone name (defaulting to `"${source.name} (Clone)"`)
- Writes `blueprint.cloned` audit with `{ sourceBlueprint, sourceName, sourceVersion, newAgentId }` metadata
- Returns new blueprint with HTTP 201

## Decisions

- **Clone creates a new logical agent (new `agentId`)**. A version increment implies the same agent evolved; a clone is a new agent derived from a parent. The clone's MRM report must not inherit the parent's reviewer/deployer records. The `sourceBlueprint` audit metadata preserves traceability.
- **Analytics computed from existing tables, no analytics store**. All needed data exists in `audit_log` and `agent_blueprints`. Aggregation queries are fast at current Intellios volumes; caching deferred.
- **Ownership PATCH endpoint (not ABP refinement)**. Organizational metadata is not AI-generated; it's structured form input. A dedicated PATCH avoids routing through the generation pipeline.
- **`deepMerge` type cast approach**. Rather than adding an index signature to `EnterpriseSettings` (which would make the type too loose), internal casts (`as Record<string, unknown>`) are used in the merge function. The public API remains fully typed.
