# Session 069 — 2026-03-29

## Objective

Conduct a full security audit of all 86 API routes against OWASP Top 10 patterns and apply immediate fixes for all actionable findings. The audit was motivated by the product reaching production-ready status (100% current product complete) — the right time to verify the security posture before expanding the user base.

## Actions

### Security Scan — 86 API Routes

All route files under `src/app/api/` were scanned for: authentication bypass, missing authorization gates, injection vectors (SQL, command, header), SSRF risks, rate-limiting gaps, and information leakage in error responses.

**Findings:** 7 (0 critical / 0 high / 3 medium / 4 low). No auth bypass or injection vulnerabilities in production code.

Full report: `docs/log/health/2026-03-29_security.md`

### SEC-001 — Cron Fails Closed (`cron/review-reminders/route.ts`)

The cron route returned HTTP 200 when `CRON_SECRET` was not set in the environment, allowing any caller to trigger the review reminder logic without authentication. Fixed to return HTTP 503 ("cron secret not configured") when the env var is absent, preventing unauthenticated execution.

### SEC-002 — Forgot-Password Rate Limit (`auth/forgot-password/route.ts`)

No rate limiting on the forgot-password endpoint allowed an attacker to enumerate email addresses via response timing or flood users with reset emails. Added IP-based rate limit: 5 requests per hour. Exceeds the limit → HTTP 429.

### SEC-003 — Webhook SSRF Prevention (`admin/webhooks/route.ts`)

`CreateWebhookSchema` accepted any HTTPS URL including RFC 1918 private ranges (10.x, 172.16–31.x, 192.168.x) and loopback (127.x, ::1). An admin could register a webhook pointing to internal infrastructure and exfiltrate data or trigger internal actions. Added a private IP regex check in the schema validation that rejects any URL resolving to a private address — HTTP 400 with `SSRF_BLOCKED` error code.

### SEC-004 + SEC-006 — npm Audit Fix

`npm audit` identified 10 vulnerabilities across the dependency tree:

- **Next.js CVEs (moderate, production):** GHSA-mq59 (null origin bypasses Server Actions CSRF), GHSA-jcc7 (null origin bypasses HMR WebSocket CSRF), GHSA-ggv3 (HTTP request smuggling in rewrites), GHSA-h27x (unbounded postponed resume buffering), GHSA-3x4c (unbounded next/image disk cache growth). Resolved via `npm audit fix`.
- **Dev-only CVEs:** flatted ≤3.4.1 (Prototype Pollution via eslint), picomatch ≤2.3.1 (ReDoS via eslint-config-next and vitest). Resolved via `npm audit fix`.

Remaining deferred: esbuild/drizzle-kit moderate CVEs require `npm audit fix --force` → drizzle-kit@0.18.1 (breaking migration upgrade). No production runtime exposure. Logged as known technical debt.

### SEC-005 — requireAuth Role Name Leakage (`lib/auth/require.ts`)

HTTP 403 responses from `requireAuth` included the required role in the response body (e.g. `"required role: compliance_officer"`). This exposed the internal role model to unauthenticated probing. Fixed to return a generic "Forbidden" message. Role names are no longer disclosed in error responses.

### SEC-007 — Reset-Password Rate Limit (`auth/reset-password/route.ts`)

No rate limiting on the password reset endpoint allowed brute-force token guessing. Added IP-based rate limit: 10 requests per hour. Exceeds the limit → HTTP 429.
