## Actions

### Phase 15: Policy-Aware Intake

#### 1. `src/lib/intake/system-prompt.ts` — Modified

Added `buildPoliciesBlock(policies: GovernancePolicy[])` function.

**Behavior:**
- Renders a new `## Active Enterprise Governance Policies` section in the system prompt
- Lists each policy with name, type, optional description, and all rules
- Each rule shows: severity tag `[ERROR]` or `[WARN]`, field path, operator, value (if any), and the human-readable message
- Instructs Claude that `[ERROR]` violations block submission and must be pre-satisfied; `[WARN]` violations should be satisfied
- Closes with field-path guidance: ensures Claude understands the mapping from rule paths to the tools that satisfy them
- Returns empty string when `policies` is empty — no section is injected if there are no policies to show

Added `policies?: GovernancePolicy[]` as a 4th optional parameter to `buildIntakeSystemPrompt`. Injected `policiesBlock` between `contextBlock` and `contributionsBlock` in the assembled prompt.

Added `import { GovernancePolicy } from "@/lib/governance/types"` at top of file.

#### 2. `src/app/api/intake/sessions/[id]/chat/route.ts` — Modified

On every chat request, after loading contributions, now also loads the enterprise's active governance policies from the `governance_policies` table — same query pattern used by `validateBlueprint()` in `validator.ts`:

```typescript
const policyRows = await db
  .select()
  .from(governancePolicies)
  .where(
    enterpriseId
      ? or(isNull(governancePolicies.enterpriseId), eq(governancePolicies.enterpriseId, enterpriseId))
      : isNull(governancePolicies.enterpriseId)
  );
```

Maps rows to `GovernancePolicy[]` and passes as fourth argument to `buildIntakeSystemPrompt`.

Added imports: `governancePolicies` from schema, `or` and `isNull` from drizzle-orm, `GovernancePolicy` and `PolicyRule` from governance types.
