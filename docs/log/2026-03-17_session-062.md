## Key Decisions

- **No new table**: `previous_blueprint_id` + `governance_diff` columns on existing `agent_blueprints`. Simpler query path; lineage is one join away; eliminates a separate versions join.
- **Diff at creation time**: governance diff computed once and stored permanently. Makes display fast (no recompute), makes it auditable (the diff is an immutable record of what changed at the moment it changed), and allows the regulatory change management question to be answered without re-deriving.
- **Patch significance for metadata-only versions**: a new version that is a direct copy of the source (before any designer edits) correctly shows "Patch — identity or metadata only" with 0 changes. This is the expected result — significance escalates as the designer modifies the draft.
- **Conditional panel display**: Version Lineage section only renders when at least one version has `governanceDiff != null`. v1.0.0 blueprints (first version of any agent) correctly have no predecessor and no lineage panel.
