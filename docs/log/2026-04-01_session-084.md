### UX Refinements (from screenshot analysis post-deploy)

Three issues identified after reviewing screenshots of the deployed redesign:

**Fix 1: Domain tag appearing on every AI message**
- Root cause: `inferActiveDomain` always returns "identity" before any tool calls (lowest-fill required domain fallback), so every pre-tool-call AI message showed `◎ PURPOSE`
- Fix: `hasToolCalls` guard in `chat-container.tsx` — `activeDomain` prop only passed to `MessageBubble` after at least one tool call has been made
- Result: no domain tag visible during initial conversation, tag appears correctly once AI starts capturing data

**Fix 2: Sidebar AWAITING SIGNAL not showing**
- Root cause: Empty state condition was `!transparency` but `transparency` gets set as soon as the first AI response arrives (even without classification data), hiding the empty state panel immediately
- Fix: Condition changed to `!transparency?.classification` — AWAITING SIGNAL persists throughout context collection until classification metadata arrives

**Fix 3: Filler word ban**
- Root cause: AI was still opening some responses with "Perfect." — existing ban was written as "Do not use..." which was insufficiently direct
- Fix: Strengthened to "Never open a response with..." in both system prompt sections; explicit list of banned phrases
