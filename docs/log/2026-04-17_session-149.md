## Decisions Made

- **Instrumentation seed without OTel dependency.** Next.js 16 invokes `register()` natively; an OTel provider can be layered later with two lines. Shipping the registration file now establishes the extension point and emits a useful boot log immediately, without committing to a tracing backend. The `@vercel/otel` upgrade path is documented in-file (ADR-022).
- **`/api/healthz` at the top level, not under `/api/monitor`.** The existing `/api/monitor/*` namespace is about per-agent deployment health (governance drift, AgentCore status). A platform liveness probe belongs at a dedicated Kubernetes-style path so uptime monitors and load balancers can hit it without traversing auth or deployment semantics.
- **Healthz is unauthenticated.** Status pages and external monitors need anonymous access. The response is deliberately sparse: no stack traces, no enterprise data, and service statuses restricted to a small enum. The value to operators outweighs the mild info-disclosure.
- **Redis "fallback" ≠ degraded.** The rate limiter has a documented in-memory fallback path. Reporting fallback as degraded would flap the endpoint every time someone runs locally without Redis. The distinction is explicit in the response shape so dashboards can render the three states differently.
- **Bedrock probe is config-only for now.** A real probe on every monitor tick would cost money and leak credit signal. Verifying `AWS_REGION` + one credential source catches the most common misconfiguration (unset env in a new environment) without spending tokens. A throttled `ListFoundationModels` probe is a logical follow-up if the team wants deeper signal.

## Deferred

- **Broader `console.*` migration (H1).** ~50 other `console.error` calls remain in non-C1–C6 routes (`blueprints/[id]/status`, `blueprints/from-template`, `intake/sessions/*`, `governance/policies/simulate`, etc.). They can migrate to `logger.error` with the same pattern used here; no design change needed. Estimate: 2–3 hours.
- **Real Bedrock reachability probe.** Current healthz only checks env presence. Upgrade path: cache a `ListFoundationModels` call for 60s and surface `"up" | "degraded" | "down"` instead of `"configured" | "not-configured"`.
- **`@vercel/otel` wiring.** Registration hook is live and safe; provider import intentionally omitted until a tracing backend is chosen.
- **Healthz auth-gated variant.** If the public surface becomes a problem, a separate `/api/admin/health` endpoint can return richer internals (connection counts, queue depths, Bedrock latency histogram).
