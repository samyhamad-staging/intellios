## Actions

### 1. Vigil loop fix

**Problem:** All Vigil scheduled tasks (`intellios-vigil-*`) write reports to `docs/log/health/` — a directory that didn't exist. Every task was silently failing (file write error), the synthesizer found no reports, and `TASK_QUEUE.md` was never produced.

**Fix:**
- Created `docs/log/health/VIGIL_STATE.json` — seeds the state file all Vigil tasks expect: `{ "lastSynthesis": null, "completedKeys": {} }`
- Created `docs/log/health/TASK_QUEUE_HISTORY.md` — required append target for the synthesizer

### 2. Agent Playground

**New function: `buildSimulationSystemPrompt(abp: ABP): string`** in `src/lib/testing/executor.ts`

Extends `buildAgentSystemPrompt` with:
- Simulation frame (cannot access real systems — prepended)
- Tool descriptions as narrative context (what the agent would do with each capability)
- Governance rules from active policies (must be followed constraints)

**New API route: `POST /api/blueprints/[id]/simulate/chat/route.ts`**

- Auth: all 4 roles. Rate limit: 30/min on `"simulate"` endpoint key.
- Fetches blueprint, asserts enterprise access, builds simulation system prompt, streams via Haiku (claude-3-5-haiku-20241022).
- Writes `blueprint.simulated` audit entry when `firstMessage: true` flag is present in the request body.
- Returns `result.toUIMessageStreamResponse()`.

**New component: `src/components/registry/simulate-panel.tsx`**

- Uses `DefaultChatTransport` (AI SDK v5 pattern) with a custom `fetch` wrapper that injects `firstMessage: true` on the first send.
- Two-component pattern: `SimulatePanel` holds a `chatKey` state; incrementing it remounts `SimulatePanelInner`, which resets `useChat` state — this is the clean way to implement "Clear conversation" without needing `setMessages`.
- Text extracted from `msg.parts` (AI SDK v5 — no `.content` field on `UIMessage`).
- Empty state: full-height centered placeholder with chat icon + description.
- Loading state: three-dot bounce animation while waiting for first assistant token.

**Registry detail page (`src/app/registry/[agentId]/page.tsx`)**

### 3. Agent Code Export

## Decisions

- **D-SIM-1:** Simulation messages are stateless (client-side only). No DB table needed. The sandbox is explicitly not a persistent conversation — it's a behavioral demonstration.
- **D-SIM-2:** `firstMessage` flag in request body (not a header) triggers the single audit entry per simulation session. Keeps the API route stateless — no session tracking on the server.
- **D-SIM-3:** Two-component pattern for "Clear conversation": outer `SimulatePanel` holds `chatKey`; incrementing it remounts `SimulatePanelInner` which fully resets `useChat` state. Clean solution without needing `setMessages` which isn't stable across AI SDK v5 minor versions.
- **D-EXPORT-1:** Code export generates a single self-contained TypeScript file (no multi-file project). Zero dependencies beyond `@anthropic-ai/sdk`. Minimal friction for developers dropping it into a project.
- **D-LANDING-1:** Unauthenticated `/` redirects to `/landing`, not `/login`. Landing page is the front door for prospects; login is for users who already have credentials.
