## Key Decisions

- **No DB schema for waitlist** — structured `console.log` sufficient for design partner tracking at this stage. Vercel log drain or Datadog/Papertrail can capture and forward. One-line DB insert swap when ready.
- **Landing page → `"use client"`** — page has no server-only features; the conversion is safe and necessary for the modal state. Next.js will still statically render at build time since there's no dynamic data fetching.
- **Render-prop pattern for modal trigger** — `RequestAccessModal` wraps the trigger via `children(open)`. This lets the same modal be triggered from different CTAs (nav button, hero button, bottom CTA) without duplicating modal state.
- **HelpPanel `"icon"` variant kept as default** — backward-compatible; `governor-sidebar.tsx` and any other places using `<HelpPanel>` without variant prop are unaffected.
