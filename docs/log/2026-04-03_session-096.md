## Key Decisions

- **`router.replace` not `push`** — Filter changes replace the current history entry rather than adding to it. This means pressing Back from a filtered view returns the user to whatever they navigated from, not to the previous filter state. This is the correct UX for filters (filters are view state, not navigation destinations). The exception would be deep links shared from outside — those land as a new entry, as expected.
- **Defaults omitted from URL** — The "agents" tab is default and not written to the URL; empty search and empty status are also omitted. This keeps the default URL clean (`/registry`) and avoids unnecessary query string noise.
- **Lazy initialization from `searchParams`** — Using the `useState(() => ...)` initializer form means the URL params are read exactly once at mount, which is correct. Subsequent navigation changes are driven by `router.replace` calls, keeping local state in sync.
- **`{ scroll: false }` on replace** — Prevents the page from scrolling to the top on every keystroke or filter click, which would be disorienting when the list is long.
