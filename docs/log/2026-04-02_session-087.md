### Implementation — P0: Ghost session prevention (quick-start-modal.tsx)

**File:** `src/components/intake/quick-start-modal.tsx` (NEW)

Created the `QuickStartModal` client component — a backdrop modal that gates session creation behind a purpose description:
- Textarea with minimum 10-character requirement + auto-resize
- 4 starter idea buttons (Customer support agent, Internal knowledge base, Data analysis & reporting, Compliance monitoring) that pre-fill the textarea
- "Start Designing" submit button — only then calls `POST /api/intake/sessions` and navigates to `/intake/{sessionId}?purpose={encodedPurpose}`
- Backdrop click to close, Escape key close, `⌘ Enter` keyboard shortcut

### Implementation — P0: Creation gate (new-intake-button.tsx)

**File:** `src/components/intake/new-intake-button.tsx` (MODIFIED)

Replaced the immediate-create pattern (`POST` on click → navigate) with modal-open pattern:
- Button click sets `modalOpen = true` → renders `<QuickStartModal onClose={() => setModalOpen(false)} />`
- Session only created inside the modal on submit
- Eliminated all `loading`, `error`, `router` state from the button — all moved into the modal
- Button label shortened: "Design a New Agent" → "New Agent" (prepares for search input in header row)

### Implementation — P1: Tab navigation + client wrapper (intake-page-client.tsx)

**File:** `src/components/intake/intake-page-client.tsx` (NEW)

Created `IntakePageClient` — a client component that receives pre-computed `SerializedSessionData[]` (Date fields as ISO strings) and handles all interactive features:

**Tab bar:** "In Progress (N)" and "Complete (N)" tabs with colored count badges. Defaults to In Progress. Eliminates the scroll-to-find-completed problem entirely.

**Client-side search:** Compact search input with Search icon. Filters by `agentName` or `agentPurpose` string match on the active tab. No server round-trip.

**Session grouping:**
- `realActive`: `hasContext === true || filledDomains > 0` — shown in "In Progress" tab
- `ghostSessions`: `!hasContext && filledDomains === 0 && status === "active"` — collapsed section "Empty — N" with one-click "Clean up N" bulk delete

**Ghost cleanup:** `GhostSessionsSection` — collapsible (default closed), header shows session count and "Clean up N" button. On click, runs `DELETE /api/intake/sessions/{id}` for each ghost in parallel, then `router.refresh()`.

**Staleness signal:** `isStale = diffDays > 7 && status === "active"`. Stale sessions show an amber `Inactive Xd` badge (replacing "In Progress") and absolute date format (replacing relative).

**Continue affordance:** Sessions with `hasContext || filledDomains > 0` show a `Continue →` label in the right metadata area that underlines on group hover.

**Domain strip improvement:** Bar dimensions increased from `h-1.5 w-2.5` → `h-2 w-4`. `x/7 domains` count label shown for ALL sessions (previously only context-bearing sessions).

### Implementation — P1: page.tsx server component refactor
