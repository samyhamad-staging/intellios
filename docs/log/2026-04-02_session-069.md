### Implementation — P0: Right panel live data + panel redesign

**File:** `src/components/intake/intake-progress.tsx`

- Renamed panel header from "Blueprint Progress" → **"DESIGN INTELLIGENCE"** with a sparkle SVG icon
- Implemented two-state design:
  - **Analysing state** (no sections filled): bouncing dots animation, "ANALYSING" label, "Insights appear as your agent design takes shape." subtitle, and a ghost checklist at 30% opacity showing what will be captured. Animation resolves the moment the first tool call fires.
  - **Live state** (any section filled): full section checklist with violet `bg-primary` fill indicators, percentage progress bar, agent name, and readiness score. Icon transitions from `text-text-tertiary` → `text-primary`.
- Added `SectionSummary` exported interface so the page can type the synced payload state.
- Added `onSectionsChange` callback prop (called on every payload refresh) using a stable `useRef` pattern to avoid infinite re-render loops from unstable function references in the effect dependency array.
- Changed progress bar from `bg-blue-500` → `bg-primary` (violet) for brand consistency.
- Fixed readiness duplication (was showing "Ready to finalize" in both the readiness card and a second status card below it).

### Implementation — P0: Classification badge always visible

**File:** `src/app/intake/[sessionId]/page.tsx`

- Risk tier badge moved from the separate floating classification bar (which could be clipped by shell redesigns) into the unified header Row 1, alongside the breadcrumb. Now rendered in all states: loading pulse while classification fires, read mode with Override link, edit mode with inline selects.

### Implementation — P1: Live domain nav bar

**File:** `src/app/intake/[sessionId]/page.tsx`

- Added `DOMAIN_NAV` constant mapping the 7 ABP sections to user-facing labels (Purpose / Capabilities / Behavior / Knowledge / Guardrails / Governance / Audit).
- Added `liveSections: SectionSummary[]` and `liveAgentName: string | null` state, synced from `IntakeProgress` via `handleSectionsChange` (wrapped in `useCallback` with empty deps).
- Row 2 of the unified header: renders 7 domain pills, each with a status dot — `bg-border` (empty), `bg-amber-400 animate-pulse` (required+unfilled), `bg-primary` (captured). Counter `X/7 domains` turns green at completion.

### Implementation — P2: Agent name breadcrumb

**File:** `src/app/intake/[sessionId]/page.tsx`

- Breadcrumb updated from "Intellios / Agent Intake" → "Design Studio › [agent name]".
- Breadcrumb is wired to `liveAgentName` state: shows "New session" (muted) until `set_agent_identity` fires, then transitions to the captured agent name with full `text-text` weight. Transition uses `duration-300` for smoothness.

### Implementation — P2: Phase 1 context summary banner

**File:** `src/app/intake/[sessionId]/page.tsx`

- Fills the previously empty whitespace in early-stage conversations, anchors the session to the Phase 1 decisions already made.

### Implementation — P3: Context-adaptive placeholder
