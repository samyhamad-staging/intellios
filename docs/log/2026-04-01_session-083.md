### Intake Chat UI Fixes (from screenshot review)

Three issues identified from screenshot evaluation:

**Issue 1: Domain strip invisible on load**
- Root cause: `DomainProgressStrip` shows "Start describing your agent" when `transparency?.domains` is empty; `transparency` is `null` until first AI response
- Fix: Added `initialDomains?: DomainProgress[]` prop to `DomainProgressStrip`; strip uses `transparency?.domains ?? initialDomains ?? []`

- Root cause: `IntakeProgress` sidebar contained old Context/Requirements/Review numbered stepper (leftover from eliminated Phase 1 form), Blueprint Progress section list + progress bar, and standalone readiness score bar — all now duplicated by the Domain Progress Strip in the header
- Fix: Full rewrite of `IntakeProgress` — removed ~430 lines of old sections, `getSections()`, `ScoreBar()`, `computeReadinessScore` usage, `fetchPayload` effect, `CONTEXT_AREAS`/`BLUEPRINT_SECTIONS` constants, and two early-return branches
- New sidebar: "Intake Insights" header using Catalyst `Subheading` + `Divider`; transparent panels only (Classification, Governance Checklist, Probing Topics, Model/Expertise indicator); Stakeholder contributions panel locked until `context` is set (not until required sections filled)
