### W2-07 — Catalyst Combobox + Checkbox

#### Part 1: Catalyst Combobox — ABP field path in Policy Form

**Rationale:** The `RuleRow` field path input in `policy-form.tsx` was a plain text `<input>`. Policy authors must know and type ABP field paths exactly (e.g., `identity.name`, `model.provider`, `tools[].name`). A Combobox with the known field suggestions dramatically reduces friction and errors — especially for new governance authors who may not have the ABP schema memorized.

**Actions:**

- **W2-07-A** — Extended `src/components/catalyst/combobox.tsx` with `onInputChange?: (query: string) => void` prop. Called alongside the internal `setQuery` in the `ComboboxInput onChange` handler. This enables free-text entry: callers update their controlled value on every keystroke (not just on option selection), making the Combobox work as a searchable suggestion list rather than a strict picker.

- **W2-07-B** — Added `ABP_FIELD_SUGGESTIONS: string[]` constant to `src/components/governance/policy-form.tsx` with 23 common ABP field paths covering all major schema sections: `identity.*`, `model.*`, `tools[].`, `resources[].`, `security.*`, `compliance.*`, `governance.*`, `constraints.*`, `metadata.*`.

- **W2-07-C** — Added `Combobox`, `ComboboxOption`, `ComboboxLabel` imports from `@/components/catalyst/combobox`.

- **W2-07-D** — In `RuleRow`, replaced the plain `<input type="text">` for the "ABP field path" field with a conditional:
  - `readOnly=true` → renders the original plain read-only input (no Combobox overhead when viewing policies)
  - `readOnly=false` → renders `<Combobox<string>>` with `ABP_FIELD_SUGGESTIONS`, free-text via `onInputChange`, and `onChange` for selection from dropdown. Placeholder `"e.g. identity.name"` preserved.

**Files modified:**
- `src/components/catalyst/combobox.tsx`
- `src/components/governance/policy-form.tsx`

#### Part 2: Catalyst Checkbox — Bulk selection on Review Queue

**Rationale:** The review queue is the highest-stakes interaction surface in Intellios. Governance officers reviewing many blueprints needed a way to bulk-assign items to themselves without opening each card. Catalyst Checkbox adoption on the review queue provides: (1) a "select all" checkbox with indeterminate state, (2) per-card Checkbox, (3) a contextual bulk action bar that shows "Assign to me" when items are selected. Assignment is the only bulk action offered — bulk approve/reject would bypass the per-item deliberation that governance requires and is out of scope for Wave 2.

**Actions:**

- **W2-07-E** — Added `UserCheck` import from `lucide-react`, `Checkbox` import from `@/components/catalyst/checkbox` to `src/app/review/page.tsx`.

- **W2-07-F** — Added `selectedIds: Set<string>` state and three helper functions:
  - `toggleSelect(id)` — toggles a single item in/out of the selection set
  - `toggleSelectAll(ids)` — selects all if not all selected, clears if all selected (true toggle)
  - `bulkAssignToMe(ids)` — assigns all selected IDs to `userEmail` via the existing `assignments` state + localStorage, then clears selection

- **W2-07-G** — Added a "bulk selection bar" just above the queue items list (shown whenever `filtered.length > 0`). Contains:
  - A "select all" `<Checkbox>` with `indeterminate` state when some (but not all) items are selected; label shows "Select all" or "X of N selected"
  - Conditional bulk action section (shown when `selectedIds.size > 0 && userEmail`): "Assign to me" button (emerald, UserCheck icon) + "Clear selection" text link
  - A 1px divider between the checkbox group and the action buttons

- **W2-07-H** — Added a per-card `<Checkbox>` inside each review queue `<Link>` card, wrapping it in a `div` with `onClick` stopPropagation to prevent card navigation when toggling. Checkbox uses `color="indigo"` and is positioned `mt-2 shrink-0` before the risk icon. Pattern matches the existing "Assign to me" button which also uses stopPropagation inside the card Link.

**Files modified:**
- `src/app/review/page.tsx`
