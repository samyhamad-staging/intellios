## Summary

12 targeted UX fixes across all four roles — Designer, Reviewer, Compliance Officer, and cross-cutting. No new routes, no migrations, no new dependencies. Eliminates the most common micro-frictions identified in the Phase 39 analysis.

## Actions

**Designer improvements (D1–D4)**

1. **D1 — Context form submit button: gray-900 → violet-600 + animated spinner.**
   - `src/components/intake/intake-context-form.tsx` — button now uses `bg-violet-600` with `inline-flex` + animated SVG spinner during submission ("Submitting…").

2. **D2 — Quality score popover dimension descriptions.**
   - `src/app/intake/[sessionId]/page.tsx` — each dimension row now shows a one-line description in `text-[11px] text-gray-400` below the progress bar. When any dimension < 3.0, an amber hint renders below the list.

3. **D3 — Classification rationale display.**
   - `src/app/intake/[sessionId]/page.tsx` — classification header now renders `classification.rationale` as italic gray text below the type/risk badge row.

4. **D4 — Pipeline Draft column empty state CTA.**
   - `src/app/pipeline/page.tsx` — Draft column with 0 cards now shows "Start an intake →" Link to `/intake` instead of bare "Empty" text.

**Reviewer improvements (R1–R3)**

5. **R1 — Review queue step badge: role-aware "Your step:" vs "Waiting:".**
   - `src/app/review/page.tsx` — added `userRole` state populated from session fetch; step badge shows violet "Your step: {label}" when `activeStep.role === userRole`, amber "Waiting: {label}" otherwise.

6. **R2 — Violations list → Governance Hub link.**
   - `src/components/review/review-panel.tsx` — added `Link` import; after the violations list, renders "View policies in Governance Hub →" link to `/compliance`.

7. **R3 — Approval History section in Versions tab.**
   - `src/app/registry/[agentId]/page.tsx` — Versions tab now includes an "Approval History" section reading `(latest.approvalProgress ?? []) as ApprovalStepRecord[]`; each step shows a color-coded dot (green=approved, red=rejected, gray=other), role, decision, and date.

**Compliance Officer improvements (C1–C3)**

8. **C1 — KPI cards as anchor links + section IDs.**
   - `src/app/compliance/page.tsx` — At-Risk, Compliance Rate, and Review Queue KPI cards converted to `<a href="#...">` anchors with hover effects; sections given `id="at-risk"`, `id="review-queue"`, `id="policy-coverage"`.

9. **C2 — At-Risk Agents subtitle.**
   - `src/app/compliance/page.tsx` — "Agents with unresolved validation errors" subtitle added below the At-Risk Agents heading.

10. **C3 — Affected agents count linked to Registry.**
    - `src/app/compliance/page.tsx` — policy coverage table's Affected Agents column now links to `/registry` when count > 0; shows "0 agents" (gray) when zero.

**Cross-cutting (X1–X2)**

11. **X1 — Null agent name fallback shows short ID.**
    - 8 files — replaced `?? "Unnamed Agent"` with `` ?? `Agent ${x.agentId.slice(0, 8)}` `` so null-name agents are identifiable: `src/app/review/page.tsx`, `src/app/pipeline/page.tsx`, `src/app/dashboard/page.tsx` (×2), `src/app/registry/[agentId]/page.tsx`, `src/app/deploy/page.tsx` (×4).

12. **X2 — AgentCore deploy success modal: View in Registry + "Done".**
    - `src/app/deploy/page.tsx` — success phase of `AgentCoreDeployModal` now shows "View in Registry →" violet button (links to `/registry/${agcModal.agent.agentId}`) alongside renamed "Done" button (was "Close").

13. **TypeScript check:** `tsc --noEmit` → 0 errors in production code.
