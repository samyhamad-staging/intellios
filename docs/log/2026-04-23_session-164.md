## OQ-010 Resolution

**Decision: Keep synthetic marker (option 2 — ship nothing).**

ADR-027 (accepted 2026-04-18) already canonically established the synthetic `[tool call simulated — invoked: X]` marker as the correct behavior for `RETURN_CONTROL` responses in Test Console. The framing is deliberate: Test Console is a *governed test harness*, not a runtime. Building a tool-mock service would:
- Blur the positioning at exactly the moment it matters most (design-partner demo)
- Add non-trivial implementation cost (fixture format, UI, schema validation, drift risk)
- Conflict with the lifecycle-demo guardrail that explicitly prohibits demonstrating RETURN_CONTROL execution

**Resolution path for enterprises that need full tool round-trips in test:** use real action-group endpoints and invoke via the Bedrock SDK directly — outside the governed Test Console.

Updated `docs/open-questions.md`: OQ-010 moved to Resolved table (2026-04-23). No ADR required — ADR-027 is the controlling document.
