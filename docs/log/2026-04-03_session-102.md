## Key Decisions

- **Rollback = navigation to registry, not a one-click API call** — The deploy page works from the "approved" list. A true rollback (re-deploying a previous version) requires choosing which version, which lives in the registry version tab. A `Link` to the registry page is the correct affordance; a dedicated "rollback" API endpoint would be premature at this stage.
- **Delta suppressed below 1%** — Sub-1% changes in invocations or error rate are within measurement noise. Showing "+0% vs prior 7d" is worse than showing nothing. `Math.abs(pct) < 1` threshold produces cleaner cards with fewer false-signal deltas.
- **prev7d uses days 8–14 window** — The natural "vs prior period" comparison for a 7-day KPI is the immediately preceding 7-day block. This gives a like-for-like weekly comparison that operators understand intuitively, vs. day-over-day which would be too granular.
- **Auto-save uses localStorage, not server-side draft endpoint** — No schema migration needed. The policy form is already client-side; localStorage is the correct persistence layer for a browser draft. The `draftKey` prop keeps it opt-in and scoped — the form component doesn't unconditionally write to localStorage.
- **Draft cleared by the page, not the form** — The form doesn't own the routing; the page does. Having the page call `localStorage.removeItem` immediately before `router.push` is the cleanest separation of concerns.
