- **H5** — Cron partial-completion handling (resume-on-restart semantics)
- **H6** — Intake prompt sanitization against prompt injection

Per Samy's direction ("all of these"), the four items are queued across sessions 152–155. The retry wrapper handles transient single-call failures; it does nothing when transience becomes the new normal. That gap is what a circuit breaker closes.

## Decisions Made

1. **Per-model breaker scope (not per-tenant, not global).** Partial Bedrock outages tend to hit specific foundation models (e.g. sonnet-4 degraded while haiku-4-5 is fine). Keying the breaker on the resolved model ID avoids over-blocking: sonnet failures don't trip haiku, and vice-versa. Per-tenant scope was considered and deferred — the failure mode we're guarding against (upstream Bedrock degradation) is provider-wide, not tenant-specific.

2. **In-memory, per-process, no Redis.** The breaker is a **reliability control**, not a **consensus mechanism**. Redis would add a dependency to the AI hot path and introduce its own failure modes (including its own circuit needs). Each Vercel function instance evaluating its own local view of Bedrock health is an acceptable approximation when the real question is "should *this* instance's next call be attempted right now?" State does not survive process restart, and that's fine — after a restart we re-probe naturally.

3. **Failure classification: skip 4xx, count 5xx/timeout/network.** A 429 "rate limited" or a 400 "bad request" is not evidence of upstream degradation. Tripping on those would over-block. `CircuitOpenError` itself is also skipped (otherwise a tripped breaker would count its own rejections as new failures and never close).

4. **Separate primitives for streaming vs. non-streaming.** `streamText` returns synchronously — the actual Bedrock request happens on stream consumption, which outlives the function scope. `generateObject`/`generateText` complete before return. Rather than force a single abstraction, we expose three primitives: `ensureCircuitClosed` (pre-flight, returns or throws), `recordBreakerSuccess`/`recordBreakerError` (out-of-band outcome reporting from `onFinish`/`onError` callbacks), and `withBedrockBreaker` (convenience wrapper tying all three together for the non-streaming case).

5. **`SERVICE_DEGRADED` 503 + `Retry-After`, not 502.** The existing `AI_ERROR` (502) code means "Bedrock tried and failed." Circuit-open means "we didn't even try — Bedrock looks degraded and we're shedding load." That's semantically 503 (service temporarily unavailable), and the `Retry-After` header lets load balancers and well-behaved clients back off without polling.

6. **`aiError` auto-routes `CircuitOpenError` → 503.** Not every catch block uses `aiError` — some use `apiError(INTERNAL_ERROR)`. To avoid polluting every route with `if (err instanceof CircuitOpenError)` branches, `aiError` now handles it internally. Routes that don't already use `aiError` got a one-line guard at the top of their catch.

## Deferred / Follow-up

- **Per-tenant breakers.** Tracked as potential future work if we ever see Bedrock region-sharding issues affecting specific tenants differently. Current per-model scope is sufficient for the failure modes we're actually seeing.
- **Manual admin endpoint to force-open/close breakers.** Useful for on-call triage but not required for the core reliability guarantee. ADR-023 lists it as explicitly deferred.
- **Metrics export (Prometheus/StatsD).** Log-based observability (`ai.breaker.{opened,half_open,closed,blocked}` events) is sufficient for now. Metrics would be a straightforward add later.
- **H5 (cron partial-completion) → session 153.**
- **H6 (intake prompt sanitization) → session 154.**

## Summary

Session 152 closed the largest AI-layer reliability gap in the production-readiness review: sustained Bedrock degradation no longer translates to cascading request failures across the platform. The breaker is conservative by default (5 failures in 30s trips a 60s cooldown, backing off exponentially to 5 minutes), per-model scoped so a sonnet outage can't starve haiku traffic, and visible to infra through `/api/healthz`. All 10 Bedrock call sites are now instrumented. Full suite 587/587.
