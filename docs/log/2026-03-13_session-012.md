## Actions

### 1. `src/app/api/intake/sessions/route.ts` — GET list endpoint

Added `GET` handler to the intake sessions collection route. Scope rules:
- Designer: sees only own sessions (`createdBy = user.email`) within their enterprise
- Admin: sees all sessions within their enterprise

Returns sessions sorted by `updatedAt` desc. Derives `agentName` from `intakePayload.identity.name` and `agentPurpose`, `deploymentType`, `dataSensitivity` from `intakeContext`. Returns `hasContext` flag for UI rendering.

### 2. `src/app/intake/page.tsx` — NEW: intake session list

Server component. Two sections: "In Progress" (active) and "Completed". Each row links to `/intake/{id}` and shows:
- Agent name (from payload) or "Untitled agent" / "New session" fallback
- Status badge (In Progress / Complete / Abandoned)
- Agent purpose (truncated)
- Context chips: deployment type + data sensitivity
- Relative timestamp + creator email

Empty state with CTA when no sessions exist. "Design a New Agent" button (`NewIntakeButton`) in header.

### 3. `src/app/layout.tsx` — Intake nav link

Added "Intake" link for `designer` and `admin` roles, positioned before "Pipeline" in the nav. Gives designers direct access to their session list from any page.

### 4. `src/lib/mrm/types.ts` — `stakeholderCoverageGaps` field

Added `stakeholderCoverageGaps: string[] | null` to `MRMReport`. Empty array = full coverage. Null = pre-Phase 8 blueprint (no intake context available for derivation).

### 5. `src/lib/mrm/report.ts` — Coverage gap assembly

Imported `getMissingContributionDomains` from `src/lib/intake/coverage`. After assembling `stakeholderContributions`, computes coverage gaps by mapping contribution rows to `StakeholderContribution[]` shape and calling `getMissingContributionDomains(intakeContext, contributions)`. Result included in `stakeholderCoverageGaps`. Null when no intake context is available.
