# Session 019 — 2026-03-13

## Summary

Phase 16: Intake Quality Gates. Closes the two silent failure modes identified in the intake confidence assessment: (1) requirements discussed conversationally but not captured in a tool call, and (2) governance policies that are structurally present but operationally vague. Two complementary mechanisms: a hard capture-verification gate at finalization, and a warning-level policy quality assessment surfaced in Phase 3.

## Actions

### 1. `src/lib/types/intake.ts` — Modified

Added three interfaces:
- `AmbiguityFlag` — moved from local definition in `intake-review.tsx` to canonical type
- `CaptureVerificationItem` — `{ area, mentioned, capturedAs: string | null }` — one entry per significant requirement Claude heard; `null` = not captured
- `PolicyQualityItem` — `{ policyName, adequate: boolean, reason }` — Claude's quality rating per governance policy; `adequate=false` is a reviewer warning, not a blocker

Added three optional underscore-prefixed fields to `IntakePayload`:
```typescript
_flags?: AmbiguityFlag[];
_captureVerification?: CaptureVerificationItem[];
_policyQualityAssessment?: PolicyQualityItem[];
```

### 2. `src/lib/intake/system-prompt.ts` — Modified

In `buildContextBlock`, added a **"Per-type quality standards"** block after the existing substance requirement note. Defines minimum adequacy per policy type:
- **safety**: must name specific prohibited behaviors — "agents must be safe" is not adequate
- **compliance**: must reference the specific regulation and what the agent must/must not do
- **data_handling**: must specify data categories and retention/deletion/masking rules
- **access_control**: must specify who is authorized and the enforcement mechanism
- **audit**: must specify retention period and whether logging is enabled

Added a closing note that `mark_intake_complete` now expects `captureVerification` and `policyQualityAssessment` fields.

### 3. `src/lib/intake/tools.ts` — Modified

**`checkGovernanceSufficiency` signature extended:**
```typescript
function checkGovernanceSufficiency(
  payload: IntakePayload,
  context: IntakeContext | null | undefined,
  captureVerification?: CaptureVerificationItem[]
): Array<{ type: string; reason: string }>
```
Added capture verification gate at the end: any item with `capturedAs === null` adds an error gap that blocks finalization.

**`mark_intake_complete` tool schema expanded:**
- `captureVerification: CaptureVerificationItem[]` (required) — Claude must enumerate every significant requirement and confirm capture
- `policyQualityAssessment: PolicyQualityItem[]` (required) — Claude must rate each policy's operational specificity

**`mark_intake_complete` execute handler updated:**
- Passes `captureVerification` to `checkGovernanceSufficiency` — uncaptured items block finalization
- `policyQualityAssessment` items with `adequate=false` are warnings only (not blocking)
- Both arrays persisted to payload via `updatePayload` before `finalizeSession()` so Phase 3 can surface them

**`flag_ambiguous_requirement` tool:**
- Cleaned up: removed old `(p as Record<string, unknown>)._flags` cast; now uses `p._flags ?? []` directly since `_flags` is properly typed on `IntakePayload`

**Imports:** Added `CaptureVerificationItem`, `PolicyQualityItem` from `@/lib/types/intake`.

### 4. `src/components/intake/intake-review.tsx` — Modified

**Imports:** Added `AmbiguityFlag`, `CaptureVerificationItem`, `PolicyQualityItem` from `@/lib/types/intake`. Removed locally-defined `AmbiguityFlag` interface.

**State:** Added `expandedCapture` boolean for the capture verification panel toggle.

**Derived state:** `captureVerification`, `policyQualityAssessment`, `inadequatePolicies` extracted from payload.

**`_flags` cast removed:** Now uses `payload._flags ?? []` directly.

**Two new panels added (before section review cards):**

1. **Capture Verification panel** — shown when `captureVerification.length > 0`. Collapsed by default with a "Show details" toggle. Table showing: area | what was discussed | captured as (green monospace) or "Not captured" (red). Subtitle explains this confirms Claude captured everything before finalizing.

2. **Policy Quality Warnings panel** — shown only when `inadequatePolicies.length > 0` (amber border). Lists each inadequate policy with a red ✗, policy name, and Claude's rationale. Note: "You may proceed, but reviewers will see these warnings."
