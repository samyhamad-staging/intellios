### New component: `FleetGovernanceDashboard`

`src/components/dashboard/fleet-governance-dashboard.tsx` — async server component:

- **Query**: joins `agentBlueprints` with `intakeSessions` via `sessionId` to get `riskTier`; filters to `approved | deployed` status; enterprise-scoped (admin sees all, others filtered by `enterpriseId`)
- **Risk tier derivation** (`deriveRiskTier()`): uses intake `riskTier` when available; falls back to governance policy-type logic (safety + compliance → high, one → medium, neither → low) for older agents without intake context
- **Governance health derivation** (`deriveGovernanceHealth()`): parses `validationReport.violations` — any error → "error", any warning → "warning", all-pass → "pass", no report → "unvalidated"
- **Renders**:
  1. 4 risk tier KPI cards (Critical / High / Medium / Low counts, color-coded)
  2. Governance alert chips: overdue for review, with governance errors, not yet validated
  3. Per-agent fleet table (Agent name + version + status + policy count | Risk Tier badge | Governance health | Next Review date or "Overdue" | Evidence link)
- Empty state: renders a dashed placeholder when no approved/deployed agents exist

## Decisions

- **Server component, direct DB query**: no intermediate API route needed — the admin Overview already uses direct DB queries for pipeline counts. Consistent pattern, lower latency
- **Risk tier fallback to policy-type logic**: seeded agents were created before Phase 43 classification (all have `null` intake `riskTier`). The policy-coverage fallback matches the MRM report derivation logic exactly — consistent tier labeling across the system
- **Evidence column links to `/blueprints/[id]/report`**: the compliance report page (not the evidence package endpoint) is the right target for the CRO — it's human-readable HTML with print-to-PDF, not a raw JSON download
- **Admin sees all agents (no enterprise filter)**: admin role bypasses the `enterpriseId` filter to give a true fleet-wide view. Non-admin roles see their enterprise's agents only
- **No new data model**: all data existed. Fleet governance is a query + presentation layer over the existing governance infrastructure
