### Audit Log Pattern

All 8 routes follow the standardized pattern:

```typescript
try {
  await db.insert(auditLog).values({
    actorEmail: authSession.user.email!,
    actorRole: authSession.user.role!,
    action: "entity.action_name",
    entityType: "entity_type",
    entityId: entityId,
    enterpriseId: enterpriseId ?? null,
    fromState: { /* optional state snapshot */ },
    toState: { /* optional state snapshot */ },
    metadata: { /* contextual details */ },
  });
} catch (auditErr) {
  console.error(`[${requestId}] Failed to write audit log:`, auditErr);
}
```

This ensures:
- Audit failures never block the primary operation (wrapped in try/catch)
- Each action captures the actor (email, role), target entity, and contextual metadata
- State transitions are recorded with fromState/toState where applicable
- Sensitive fields are not logged as plaintext (e.g., SSO secret changed is boolean, not value)
