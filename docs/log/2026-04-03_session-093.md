## Key Decisions

- **`HealthPulse` derives from `violationCount` + `warningCount`** — no additional API call or new DB column needed. Both fields derived from existing `validationReport` JSONB.
- **Context bar does not replace `LifecycleControls`** — the header buttons remain as the authoritative action point. The bar adds _context_ (why to act) and a convenience CTA; `LifecycleControls` is the fallback if the bar's fetch fails.
- **Inline bar for `deployed` status skipped** — the existing Governance Health Strip already provides rich deployed-state UX. Adding another bar would duplicate it.
- **Fix guidance links use hash anchors for settings** (e.g., `/admin/settings#governance-rules`) — consistent with the section IDs added in Session 092's sticky nav.
