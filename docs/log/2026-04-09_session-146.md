# Session 146 — 2026-04-09

**Evidence Package PDF — Sample Rendering for Regulatory Delivery**

## Context

Samy asked for a 10-page PDF that looks exactly like a Big-4 audit deliverable an enterprise CRO would receive after running an AI agent through Intellios's governance pipeline. Target fictional agent: Claims-Triage-Agent v2.1. Target fictional customer: Acme Mutual Insurance (a P&C insurer). Target outcome: an artifact that can be shown to investors, design partners, and regulators without being technically wrong about what the product does.

The first draft of the outline was audited against the actual codebase and ~35% of it turned out to be fabricated (pre-execution policy gate, cryptographic attestation with SHA-256 + Merkle root, per-decision runtime audit trail). The real evidence-package export in `src/app/api/blueprints/[id]/evidence-package/route.ts` emits JSON, not PDF, and the real `MRMReport` has 14 sections — not the 10 I originally outlined.

After presenting the verdict (real / aspirational / fabricated) Samy selected **Path B**: faithful current-state output built from the real `MRMReport` shape, with Option 1 framing ("sample rendering of the real schema, customer data illustrative"). Samy also made a product commitment: Intellios must have the capability to emit this PDF from real data — this is now a product requirement, not a mockup-only deliverable.

## Actions

### 1. Revised outline to match real `MRMReport` + evidence-package wrapper

Mapped every PDF element to a real field in `src/lib/mrm/types.ts` or the wrapper produced by `src/app/api/blueprints/[id]/evidence-package/route.ts`. Dropped fabricated items: pre-execution policy gate, cryptographic attestation, per-decision runtime audit, HITL-rate/decisions-per-day metrics, "98/100 governance score." Added real fields initially skipped: `qualityEvaluation` (5-dimension rubric) and `testEvidence` (blueprint harness run). Rewrote the "something got caught" narrative to use the real mechanism: blueprint v2.0 was blocked by design-time governance validation (3 error-severity violations against Safety Baseline), then remediated as v2.1.

### 2. Built a reportlab generator with hand-written seed data

`samples/build_evidence_pdf.py` — ~1400 lines of Python that:
- Defines a brand palette pulled from `docs/design-tokens.md` (indigo `#4f46e5`, WCAG-compliant text scale, emerald / amber / red for status, slate ruleset).
- Hand-writes a `SEED` dict that mirrors the full evidence-package shape: `exportMetadata`, `cover`, `riskClassification`, `identity`, `capabilities`, `governanceValidation`, `reviewDecision`, `sodEvidence`, `approvalChain`, `modelLineage`, `deploymentRecord` (with `agentcoreRecord`), `regulatoryFrameworks` (3 frameworks × real requirement IDs from `src/lib/regulatory/frameworks.ts`), `auditChain` (8 lifecycle events), `stakeholderContributions`, `workflowContext`, `periodicReviewSchedule`, `qualityEvaluation` (5 dimensions), `testEvidence`.
- Every enterprise-specific fabricated value is tagged with an inline `[FR]` (founder review) badge.
- `BaseDocTemplate` with two `PageTemplate`s: a cover template with top indigo bar + bottom classification strip, and a content template with running header, thin rule, page number footer, and sample-rendering disclaimer.
- Helper primitives: `kv_table`, `data_table` (indigo header + alternating row backgrounds + thin rules), `yaml_block` (line-numbered code blocks with slate-100 background), `stat_box`, `callout`, `section_header`.
- Page builders for 10 sections: Cover, Executive Summary, Identity & Capabilities, Policy-as-Code, Governance Validation (with v2.0 historical context), Review Decision + SOD + Approval Chain, Model Lineage + Deployment + AgentCore record, Regulatory Framework Assessment (EU AI Act / SR 11-7 / NIST AI RMF), Quality Evaluation + Test Evidence, Audit Chain + Stakeholders + Workflow + Periodic Review, Formal Attestation.

### 3. Iterated on reportlab bugs

First pass failed on `ValueError: Invalid color value 'b45309'`. Root cause: `Color.hexval()[2:]` returns a hex string without the `#` prefix, but reportlab's inline `<font color=...>` markup requires the `#`. Added an `hx()` helper that always returns `#RRGGBB`, replaced all six call sites across `stat_box`, `callout`, `build_lineage_deployment`, `build_regulatory_frameworks`, and `build_quality_and_tests`.

### 4. Verified the generated PDF

Rendered to PNG via `pdftoppm -r 110 -png` and visually inspected every page. Findings:
- 14 pages total (up from the 11 I estimated — content denser than projected, but clean breaks throughout).
- No Unicode glyph box artifacts (all sub/super tags, no Unicode sub/superscript characters).
- Tables, YAML code blocks, stat boxes, chips, and signature blocks all render as intended.
- Running header shows agent + enterprise + export timestamp. Page footer has page number, "Sample rendering v1.0", and "Intellios / Acme Mutual Insurance [FR]".
- Cover page has the top indigo wordmark bar and the bottom `CONFIDENTIAL — REGULATORY EVIDENCE` classification strip.
- Signature page at the end renders three formal attestation blocks (architect, reviewer, deployer) sourced from the real `sodEvidence` shape, with an honest package-integrity statement explaining that Intellios does not currently emit cryptographic signatures.
- Page 13 has a lot of orphan whitespace because Section 10 forces a page break — cosmetic only, not a layout bug.

## Summary

Delivered a 14-page sample PDF rendering of the Intellios evidence package for Claims-Triage-Agent v2.1 / Acme Mutual Insurance. Every field maps to a real `MRMReport` or wrapper field; fabricated enterprise data is flagged `[FR]`. The sample is reproducible via `samples/build_evidence_pdf.py` and is positioned in marketing and design-partner conversations as "a sample rendering of the real Intellios evidence-package schema v1.0 as of 2026-04-09, with illustrative customer data."
