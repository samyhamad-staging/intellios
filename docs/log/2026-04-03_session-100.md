## Key Decisions

- **"Review next critical" uses `queue[0]`** — The queue is fetched from `/api/review` and rendered as-is. The first item is the highest-priority one (the API returns items sorted by SLA/risk). The CTA links directly to `?tab=review` on that agent's registry page. Only rendered when the queue is non-empty.
- **Governor banner as server component wrapper** — Replacing re-exports with wrapper components (no `"use client"`) keeps server-side rendering for the banner strip itself. The imported original pages remain client components within the server component tree — this is the standard Next.js App Router pattern. The banner is a simple styled div with no interactivity needed.
- **ReviewSlaBadge shows 3 states** — gray (safe), amber (< 24h remaining), red (overdue). `REVIEW_SLA_DAYS = 5` constant at the top of the file for easy configuration. The badge is placed above the AI Risk Brief so reviewers see urgency context before generating a brief.
- **Waiting time in the governor CTA is computed inline** — Reusing the `SLARemaining` component would have applied wrong color classes in the violet context. Inline IIFE computes the same display but styled as violet-on-violet.
