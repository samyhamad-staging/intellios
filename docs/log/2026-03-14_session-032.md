# Session 032 — 2026-03-14

## Summary

Phase 28: Awareness and Measurement System — real-time quality scoring, SQL metrics aggregation, anomaly detection, daily AI-synthesized intelligence briefings, and the Trend Observatory UI at `/monitor/intelligence`.

---

## Actions

### 1. Architecture design session (prior context)
Designed the three-question system: generation correctness, quality trend, reliability/safety. Established event-driven side-effect pattern for quality scoring, SQL-only metrics snapshots, AI briefing via Claude Sonnet.

### 2. Database migration — `0014_awareness_system.sql`
Four new tables:
- `blueprint_quality_scores` — AI-evaluated per-blueprint quality dimensions (5 numeric scores + flags)
- `intake_quality_scores` — AI-evaluated per-session intake quality (4 dimensions)
- `system_health_snapshots` — periodic SQL aggregation metrics + quality index composite
- `intelligence_briefings` — daily AI narrative with health status; UNIQUE on `(enterprise_id, briefing_date) NULLS NOT DISTINCT`

### 3. Schema extension — `src/lib/db/schema.ts`
Added 4 Drizzle table definitions with `numeric`, `date`, and `jsonb` columns.

### 4. Types — `src/lib/awareness/types.ts`
New exports: `MetricsSnapshot`, `QualityScoreResult`, `IntakeScoreResult`, `AnomalySignal`, `BriefingResult`, `HealthStatus`, `IntelligencePayload`.

### 5. Metrics worker — `src/lib/awareness/metrics-worker.ts`
`runMetricsSnapshot(enterpriseId)`: 8 SQL aggregations (validity rate 7d, avg refinements, review queue depth, SLA compliance, webhook success 24h, policy count, blueprints 24h, violations 24h). Quality Index formula: `validityRate×40 + refinementEfficiency×20 + slaRate×20 + webhookRate×10 + (policyCount>0?10:0)`. Writes to `system_health_snapshots`. Also exports `getRecentSnapshots()`.

### 6. Anomaly detector — `src/lib/awareness/anomaly-detector.ts`
`detectAnomalies(snapshot, previous)`: threshold checks on 4 signals (validity rate, quality index drop, webhook success, queue depth). Deduplicates by querying `notifications` table for same type in last 24h. Calls `createNotification()` for compliance_officers + admins.

### 7. Quality evaluator — `src/lib/awareness/quality-evaluator.ts`
Side-effect module. Registers one handler covering `blueprint.status_changed` (→ `in_review`) and `intake.finalized`. Uses `claude-haiku-4-5-20251001` with `temperature: 0`. Blueprint scoring: 5 dimensions (1–5) + flags → `overallScore = mean × 20`. Intake scoring: 4 dimensions. All errors caught, never throws. Wired into `src/lib/audit/log.ts` via side-effect import.

### 8. Briefing generator — `src/lib/awareness/briefing-generator.ts`
`generateDailyBriefing(enterpriseId, date)`: gathers 2 snapshots, 5 recent quality scores, 24h audit activity. Calls `claude-sonnet-4-6` with `temperature: 0.2`. 5-section structured narrative. Upserts via `onConflictDoUpdate`. Pushes notification if health ≠ nominal. Fallback to `buildFallbackBriefing()` on AI failure.

### 9. API routes (3 new files)
- `POST /api/monitor/intelligence/snapshot` — run metrics + anomaly detection
- `POST /api/monitor/intelligence/briefing` — generate briefing (admin)
- `GET /api/monitor/intelligence/briefing` — last 7 briefings (compliance_officer | admin)
- `GET /api/monitor/intelligence` — combined KPI + snapshots + quality scores + briefing

### 10. Intelligence page — `src/app/monitor/intelligence/page.tsx`
Client component. Header with Refresh Snapshot + Generate Briefing (admin) + ← Monitor. 4-card KPI strip (Quality Index + delta, Validity Rate, Review Queue, Webhook Rate). Latest briefing panel with health badge. SVG sparkline (no chart library). Recent blueprint quality scores table.

### 11. Settings extension
`awareness` block added to `EnterpriseSettings`: alert thresholds (4 values) + optional `briefingWebhookUrl`. Defaults wired in `get-settings.ts`.

### 12. Monitor nav link
`/monitor/intelligence` link added to `/monitor` page header.

### 13. TypeScript fixes
Corrected `ErrorCode.INTERNAL` → `ErrorCode.INTERNAL_ERROR` (3 route files). Removed invalid `maxTokens` property from 3 `generateText()` calls. Final TypeScript check: 0 errors.

### 14. Scheduled task
Created `intellios-daily-briefing` scheduled task: cron `0 8 * * *` (8:00 AM daily), POSTs to `/api/monitor/intelligence/briefing`.
