## Guided Workflow Creation Wizard

Created `src/components/workflow/creation-wizard.tsx` тАФ a 5-step dialog component that replaces the minimal "New Orchestration" dialog on the Registry page.

**Steps:**
1. **Basics** тАФ Name, description, and optional template selection (loads from `/api/workflows/templates`). Templates populate agents, handoff rules, and shared context.
2. **Agent Selection** тАФ Browse and add approved/deployed agents from the registry. Each agent can have its role renamed and be toggled required/optional.
3. **Handoff Rules** тАФ Define control flow between agents. Auto-scaffolds a sequential chain (start тЖТ A1 тЖТ A2 тЖТ ... тЖТ end) when entering the step. Each rule has from/to selectors, a condition field (defaults to "always"), and priority.
4. **Shared Context** тАФ Add context fields (field path, type, description) that agents read/write during orchestration. Optional step.
5. **Review** тАФ Summary of all configuration with a governance warning if all rules are unconditional.

**Integration:** Registry page (`src/app/registry/page.tsx`) updated to use `WorkflowCreationWizard` instead of the simple name/description dialog. Old dialog state variables (`newOrchName`, `newOrchDesc`, `creatingOrch`, `orchCreateError`, `handleCreateOrchestration`) removed. Unused `Dialog`-related Catalyst imports cleaned up.

## Inline ABP Field Editing

### Component: `src/components/blueprint/inline-field-editor.tsx`

Two exports:
- **`InlineFieldEditor`** тАФ Click-to-edit component supporting 7 field types: text, textarea, number, boolean, select, tags (array editor), and json (with validation). Shows pencil icon on hover, transitions to edit mode with save/cancel controls.
- **`EditableRow`** тАФ Convenience wrapper combining a label with `InlineFieldEditor` to match the existing `Row` pattern in BlueprintView.

Sub-components: `TagEditor` (add/remove tags with enter key), `JsonEditor` (textarea with real-time parse validation).

### API: `src/app/api/blueprints/[id]/fields/route.ts`

PATCH endpoint for individual field updates. Features:
- Dot-path addressing (e.g. `identity.branding.color_primary`)
- Whitelist of editable prefixes: `identity.*`, `capabilities.*`, `constraints.*`, `governance.*`, `ownership.*`, `execution.*`
- Draft-only gate тАФ returns 400 for non-draft blueprints with guidance to use the refinement chat
- Returns previous and new values plus the full updated ABP

### BlueprintView integration (`src/components/blueprint/blueprint-view.tsx`)

- Extended `BlueprintViewProps` with optional `blueprintId` and `onFieldSaved` callback
- Added `canEdit` flag (true when `status === "draft"` and `blueprintId` is provided)
- Identity section: name, description, persona, display name, primary color тАФ all editable inline for drafts
- System Instructions section: editable textarea for drafts; also shown when empty in edit mode (previously hidden)
- Constraints section: allowed domains (tags), denied actions (tags), max tokens (number), rate limits (numbers) тАФ all editable inline
- "Inline editing enabled" badge shown in summary card header for draft blueprints
- Non-draft blueprints retain the original read-only rendering unchanged

## Decisions

- Inline editing is restricted to draft-status blueprints only тАФ architects must use the refinement chat for in-review or approved blueprints, preserving the governance audit trail.
- Editable field paths are whitelisted by prefix to prevent accidental corruption of metadata or version fields.
- The creation wizard auto-scaffolds a sequential handoff chain when entering step 3, reducing friction for the most common orchestration pattern.
- Template selection on step 1 pre-populates all subsequent steps, allowing quick starts from established patterns.
