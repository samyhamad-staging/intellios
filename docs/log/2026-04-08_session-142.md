## Findings

- **Thenable mock pattern required:** Governance and auth routes frequently use bare `await db.select().from().where()` without calling `.limit()`. The Session A mock-db assumed all select chains end with `.limit()`. The fix was to make `where()` return a thenable object that resolves to `selectResult()` when awaited, while still supporting `.limit()` for chains that use it.
- **Node crypto module aliasing:** `import { randomUUID } from "crypto"` in route files resolves to `node:crypto` at runtime. Mocking `crypto` alone is insufficient — `node:crypto` must also export the same symbols.
- **Insert chain extensibility:** The register route uses `.returning({...}).onConflictDoUpdate({...})` for enterprise settings, requiring mock extension beyond the Session A patterns.
