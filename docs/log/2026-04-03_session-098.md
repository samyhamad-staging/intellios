## Key Decisions

- **Auto-mark-on-open removed** — Opening a notification dropdown to browse should not silently mark everything read. Explicit "Mark all as read" button gives users agency. The optimistic update pattern is preserved (immediate visual feedback, PATCH in background).
- **Analytics collapsed by default** — The governance page is 1000+ lines and most users visit it to act (create policy, check violations), not to read charts. Collapsing saves ~300px of prime scroll real estate. The summary line gives instant signal without expanding.
- **Summary line uses JSX span for violations** — The red violation count needed a different color inline. Used an immediately-invoked function expression in the JSX to conditionally render the red span.
- **Quality gate is advisory only** — Blocking submission on quality score would create friction for edge cases (e.g., the first blueprint before the quality evaluation runs). Advisory warning with a link to the Quality tab surfaces the concern without preventing action. `QUALITY_GATE_THRESHOLD = 3.0` is defined as a constant for easy tuning.
- **Quality fetch gated by `qualityGateLoaded` flag** — Same pattern as `testDataLoaded` to prevent double-fetching on React re-renders.
