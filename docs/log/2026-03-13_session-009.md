## Summary

**Three-phase enterprise intake architecture** — eliminates the governance blindspot where Claude
could not know to ask about domain-specific compliance requirements without explicit user prompting.

### Phase 1 — Structured Context Form
New `IntakeContextForm` component captures 6 domain signals before the AI conversation begins:
agent purpose, deployment type, data sensitivity, regulatory scope, integration types, and
stakeholders consulted. Submitted via `PATCH /api/intake/sessions/[id]/context`. Session page
gates to Phase 2 only after this form is complete.

### Phase 2 — Guided AI Conversation (enhanced)
System prompt now receives `IntakeContext` and injects two new sections:
- **Enterprise Context block** — frames Claude's opening message and prevents re-asking what the user already submitted
- **Mandatory Governance Probing Rules** — deterministic list of what must be captured based on context signals (5 trigger rules)

`mark_intake_complete` now enforces a governance sufficiency matrix: if required policies or audit config are missing given the context, the call is rejected with a clear gap list, preventing finalization with incomplete governance requirements.

New `flag_ambiguous_requirement` tool — Claude can flag unclear requirements to `_flags` array in the payload; flagged items appear in the Phase 3 review screen for human review.

### Phase 3 — Pre-Finalization Review Screen
New `IntakeReview` component replaces the old completion banner. Shows:
- Enterprise context summary strip
- Ambiguity flags panel (expandable, with field/description/user statement)
- Per-section review cards with rich content display
- Per-section acknowledgment checkboxes (Generate button disabled until all filled sections acknowledged)
- Clear messaging when required sections are missing

### MRM Report enriched
`riskClassification` section now includes deployment type, data sensitivity, regulatory scope, and
stakeholders consulted from Phase 1 context — giving model risk officers a complete intake evidence
chain in the compliance report.
