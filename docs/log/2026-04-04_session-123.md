### W3-07: React Query Adoption

**Investigation:**

- Confirmed `@tanstack/react-query` v5.96.2 already present in `node_modules` (installed but not in `package.json`).
- Surveyed all `src/app/**/*.tsx` files for `fetch(` calls. 33 pages with fetch calls identified.
- Priority list pages by fetch count: `governance/page.tsx` (8), `monitor/page.tsx` (5), `compliance/page.tsx` (5), `blueprints/page.tsx` (1), `registry/page.tsx` (3), `review/page.tsx` (3).
- Read `src/components/providers.tsx` — existing `SessionProvider` wrapper; ideal location for `QueryClientProvider`.
- Read `src/app/layout.tsx` — confirmed `<Providers>` component wraps all authenticated pages.
- Read Drizzle schema for `governance_policies` columns to ensure fetcher types match actual API response (`type`, not `policyType`; `policyVersion`, not `version`; etc.).

**`package.json`:**
- Added `"@tanstack/react-query": "^5.96.2"` to `dependencies`.

**`src/components/providers.tsx` — QueryClientProvider:**
- Added `QueryClient`, `QueryClientProvider` imports from `@tanstack/react-query`.
- Browser singleton pattern: `getQueryClient()` returns a module-level `browserQueryClient` instance on the client; always creates fresh on the server to avoid cross-request state sharing.
- Default options: `staleTime: 30s`, `gcTime: 5min`, `retry: 1`, `refetchOnWindowFocus: true`.
- `QueryClientProvider` wraps `SessionProvider` — all child components can now call `useQuery`.

**`src/lib/query/keys.ts` — Query Key Factory:**
- Central key registry using factory functions (arrays, never plain strings).
- Domains: `registry` (agents, workflows, agent, quality), `blueprints` (all, list, detail), `governance` (policies, templates, analytics), `review` (queue), `monitor` (agents, intelligence), `compliance` (overview), `audit` (log), `pipeline` (board), `me`.
- Hierarchy: `[domain, resource, ...params]` for granular invalidation.

**`src/lib/query/fetchers.ts` — Typed Fetcher Library:**
- Central `apiFetch<T>(url)` helper: throws `Error` with API message on non-OK responses so React Query populates `error` state correctly.
- Typed interfaces: `RegistryEntry`, `WorkflowEntry`, `BlueprintEntry`, `GovernancePolicySummary` (corrected to match actual DB schema), `GovernanceTemplate`, `ReviewItem`.
- Exported functions: `fetchAgents`, `fetchWorkflows`, `fetchBlueprints`, `fetchPolicies`, `fetchGovernanceTemplates`, `fetchGovernanceAnalytics`, `fetchReviewQueue`, `fetchMonitorAgents`, `fetchPipelineBoard`, `fetchAuditLog`, `fetchMe`.

**Page Migrations (5 pages):**

1. **`registry/page.tsx`** — Replaced agents `useState+useEffect` with `useQuery(queryKeys.registry.agents(), fetchAgents)`. Replaced workflows `useState+useEffect` with `useQuery(queryKeys.registry.workflows(), fetchWorkflows, { enabled: activeTab === "workflows" })` — preserves lazy-load-on-tab-switch behavior. Removed `useEffect` import (no longer needed after migrations).

2. **`blueprints/page.tsx`** — Replaced `useState+useEffect` with `useQuery(queryKeys.blueprints.list(), fetchBlueprints)`. Removed `useEffect` import.

3. **`governance/page.tsx`** — Replaced 3 parallel initial fetches (registry, policies, templates) with 3 independent `useQuery` calls. Analytics query uses `{ enabled: canViewAnalytics, staleTime: 2*60*1000 }`. `analyticsDays` state moved before query declarations (was used in query key but declared after). `loadPolicies()` function removed; replaced with `queryClient.invalidateQueries({ queryKey: queryKeys.governance.policies() })` after template pack import. `analyticsLoadedAt` derived from `dataUpdatedAt` (React Query's timestamp). Added `useQueryClient` import. Removed `useEffect` import.

4. **`review/page.tsx`** — Complex chained pattern (session → role → queue URL) migrated to: `useSession()` for session data (role, email), then `useQuery` with role-derived URL and `enabled: sessionData !== undefined`. Admin settings fetch kept as a `useEffect` (best-effort, non-critical, triggers on role change). Added `useSession` import.

5. **`monitor/page.tsx`** — `loadAgents` function replaced with `useQuery(queryKeys.monitor.agents(), ...)` with `refetchInterval: 30_000` (matches old 30s polling pattern), `refetchIntervalInBackground: false`. `handleCheckAll` and `handleCheckOne` now call `queryClient.invalidateQueries` after mutations instead of mutating local state directly. `fetch("/api/me")` for role replaced with `useSession().data?.user?.role`. Added `useSession`, `useQueryClient` imports.
