# Session 020 ‚Äî 2026-03-13

## Summary

Phase 17: Generation Intelligence. Three targeted improvements to blueprint generation effectiveness and intake cost efficiency:

1. **`loadPolicies` shared utility** ‚Äî extracted the enterprise policy query from `validator.ts` and the inline copy in `chat/route.ts` into a single shared `src/lib/governance/load-policies.ts` module. All four call sites now use the same implementation, eliminating divergence risk.

2. **Policy-aware blueprint generation** ‚Äî `generateBlueprint` and `refineBlueprint` now receive enterprise governance policies and inject them into the generation system prompt. Claude can satisfy error-severity rules proactively during generation rather than discovering violations in the post-generation validation pass. The generate route loads policies once and passes them to both `generateBlueprint` and `validateBlueprint`, eliminating the redundant second DB query that would otherwise occur. The `validateBlueprint` signature was extended with an optional `policies?` parameter (backward-compatible: existing callers are unchanged).

3. **Adaptive model selection for intake chat** ‚Äî a new `selectIntakeModel()` function in `src/lib/intake/model-selector.ts` routes intake turns to Claude 3.5 Haiku (~75‚Äì80% of turns) or Sonnet 4 (opening, governance-heavy, and post-completion turns). The primary finalization signal is payload completeness (identity name + at least one tool filled), not keyword matching. Keyword matching handles early finalization and explicit governance discussion as secondary layers.

4. **Tool call ceiling increased** ‚Äî `stopWhen: stepCountIs(10)` raised to `stepCountIs(20)`. In AI SDK v5, this limits reasoning steps (rounds of LLM inference), not raw tool calls. 20 steps protects against sessions with many back-and-forth reasoning cycles.

5. **Haiku for remediation suggestions** ‚Äî `addRemediationSuggestions` switched from `claude-sonnet-4-20250514` to `claude-3-5-haiku-20241022`. Remediation suggestions are short, structured, factual completions per violation ‚Äî Haiku handles this reliably.

Shared enterprise policy loader. Exports `loadPolicies(enterpriseId: string | null): Promise<GovernancePolicy[]>`. Implements the `or(isNull, eq(enterpriseId))` pattern that was previously duplicated in `validator.ts` and `chat/route.ts`.

### 4. `src/lib/generation/system-prompt.ts` ‚Äî Modified

Converted from a static exported string constant to an exported function:

```typescript

```

When policies are provided, appends a `## Enterprise Governance Policies` block listing all rules with `[ERROR]`/`[WARN]` severity tags and dot-notation field paths. Instructs Claude to satisfy error-severity rules proactively during generation. When policies are absent, returns the base prompt unchanged (backward-compatible).

### 8. `src/lib/intake/model-selector.ts` ‚Äî New

Exports:
- `IntakeModel` type: `"claude-sonnet-4-20250514" | "claude-3-5-haiku-20241022"`
- `ModelSelectionContext` interface: `{ messageCount, lastUserText, context, payload }`
- `selectIntakeModel(ctx: ModelSelectionContext): IntakeModel`

Routing logic (in priority order):
1. `messageCount <= 1` ‚Üí Sonnet (opening turn)
2. `payload.identity?.name && tools.length > 0` ‚Üí Sonnet (payload complete, may finalize)
3. Explicit finalization keywords in `lastUserText` ‚Üí Sonnet
4. Governance/regulatory keywords in `lastUserText` ‚Üí Sonnet
5. All other turns ‚Üí Haiku
