## Decisions Made

- **Governance block is hard, override is audited.** A blueprint with error-severity violations cannot be approved by a reviewer. Admins may override, but only with `governanceOverride: true` + `overrideReason` (≥20 chars). The override produces a separate `blueprint.approved.override` audit row in addition to the normal review audit. This preserves the append-only audit contract while giving a controlled escape valve (ADR-019).
- **Atomic writes via explicit `db.transaction`, not via a shared helper.** Kept scope tight: inlined `tx.insert(auditLog)` inside each route's transaction rather than refactoring `writeAuditLog` to accept a `tx` parameter. Smaller blast radius, no shared-helper regressions, easy to diff in review (ADR-021).
- **Per-tenant limits default to 6× per-user limit.** Most AI endpoints have `max: 30–40` per user; enterprise ceiling is `240` (~6–8 active users/min). Blueprint generation specifically is `10` per-user / `60` per-tenant. Numbers are conservative and refinable without a schema change (ADR-020).
- **No global abort timeout on `streamText`.** Tool-calling chains legitimately span tens of seconds across multiple provider round-trips. `maxRetries: 3` covers the first-byte / transient-error case; the Vercel function `maxDuration` (60s) is the effective ceiling per turn. Revisit if we move to longer-running background jobs.
- **DB pool defaults: `max=20`, `idle=30s`, `connect=10s`.** Sized for long-running Node deployments (Docker/EC2). Serverless callers should override to 5–10 via env (ADR-017).
- **`SECRETS_ENCRYPTION_KEY` now boot-gates production.** The env Zod schema refuses production boot if the 64-hex-char key is missing. The crypto module's dev/test plaintext fallback remains for local work (ADR-018).
