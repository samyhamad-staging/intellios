# Session 044 — Demo Deployment + Settings Save Fix

**Date:** 2026-03-15
**Branch:** master

---

## Summary

Completed the public Intellios demo deployment to Vercel + Neon and fixed a production bug discovered during AgentCore settings configuration.

---

## Actions

1. **Retrieved Neon connection string** from Vercel env vars (`STORAGE_DATABASE_URL_UNPOOLED`) and updated `DATABASE_URL` to the real Neon URL via Vercel API.

2. **Updated `run-migrations.ts`** to include all 20 migrations (0000–0019) so it works against a fresh database, not just incremental updates.

3. **Ran `drizzle-kit push`** on the empty Neon DB to create the full schema (needed because migrations 0000–0003 don't create `users`/`audit_log` tables — those are in the Drizzle schema but not in early migration files).

4. **Applied all 20 migrations** on Neon production DB — all succeeded with IF NOT EXISTS clauses handling the push-created schema.

5. **Seeded demo data**: 4 users, enterprise settings, governance policies, 5 agents across lifecycle stages, audit log, deployment health, test cases, quality scores, intelligence briefing.

6. **Deployed to production** at `https://intellios.vercel.app` — build passed in ~44s.

7. **Fixed "Save failed" bug** on admin settings page:
   - **Root cause:** Seed inserted approval chain steps as `{ role, label }` without `step` field. When settings page sent them back, `ApprovalChainStepSchema` required `step: z.number().int().min(0)` → Zod rejected the payload.
   - **Fix:** Made `step` optional in schema; added normalization `step: s.step ?? i` (auto-fill from array index).
   - Redeployed; user confirmed save succeeded.

8. **AgentCore configured** in the app: `us-east-1`, `arn:aws:iam::540628117221:role/BedrockAgentExecutionRole`, `anthropic.claude-3-5-sonnet-20241022-v2:0`.
