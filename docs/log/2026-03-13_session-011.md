## Actions

### 1. `src/lib/intake/tools.ts` — Substance check added to `checkGovernanceSufficiency`

After the existing type-presence checks (does a policy of type X exist), added a second pass over required types. For each required policy type that passes the presence check, verified the policy is *substantive*: at least 1 non-empty rule in `rules[]` OR `description` trimmed length ≥ 25. Failures returned as `{ type: "${type}_substance", reason: "..." }` gaps — formatted identically by the existing `mark_intake_complete` error display.

### 2. `src/lib/intake/system-prompt.ts` — Policy substance instruction

In `buildContextBlock`, added a **Policy substance requirement** line to the governance probing rules section. Claude is now explicitly instructed to include substantive content when calling `add_governance_policy`, and told that empty shells will be rejected at finalization.

### 3. `src/lib/intake/coverage.ts` — NEW coverage helper

New file with two exported functions:
- `getExpectedContributionDomains(context: IntakeContext): ContributionDomain[]` — maps Phase 1 signals (regulatory scope, data sensitivity, deployment type, integrations, stakeholdersConsulted) to expected contribution domains. Full deduplication via `Set`.
- `getMissingContributionDomains(context, contributions): ContributionDomain[]` — returns expected domains with no contribution on record.

Signal-to-domain mappings:
- FINRA/SOX → compliance
- GDPR/HIPAA → compliance, legal
- PCI-DSS → compliance, security
- PII/regulated data sensitivity → compliance, security, legal
- Confidential data → security
- Customer/partner-facing → legal, business
- External APIs → security, it
- Databases/file-systems → it
- stakeholdersConsulted direct: legal→legal, compliance→compliance, security→security, it→it, business-owner→business

### 4. `src/components/intake/stakeholder-contributions-panel.tsx` — Coverage gap strip

Added `context?: IntakeContext` prop. When provided and `getMissingContributionDomains(context, contributions).length > 0`, renders amber coverage warning strip below the header listing missing domain chips (colored with existing `DOMAIN_COLORS`). Strip disappears automatically when all gaps are covered.

### 5. `src/components/intake/intake-review.tsx` — Missing-domain callout

Added `getMissingContributionDomains` import. The contributions section now renders even when `contributions.length === 0` if there are missing domains. After the contribution cards, an amber callout lists each missing domain with the message: "These domains were implicated by your intake context. You may proceed, but reviewers will note the absence." Non-blocking.

### 6. `src/components/intake/intake-progress.tsx` — Context prop pass-through

Added `context?: IntakeContext` to `IntakeProgressProps`. Passes `context` down to `StakeholderContributionsPanel`.

### 7. `src/app/intake/[sessionId]/page.tsx` — Context wired to IntakeProgress

Passes `intakeContext ?? undefined` to `<IntakeProgress context={...}>`.
