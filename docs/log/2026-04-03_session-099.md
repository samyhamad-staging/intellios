## Key Decisions

- **Red team export as JSON (not PDF)** — The report data is already fully structured (`RedTeamReport` type). A JSON export gives governance teams a machine-readable artefact that can be imported into audit tools or evidence packages. PDF would require a new dependency (pdfmake or equivalent) — deferred per zero-new-deps constraint. The export filename includes agent name and version for traceability.
- **RubricTooltip as click-toggle, not hover** — Hover tooltips in the DIMENSIONS list would overlap other rows and are inaccessible on touch devices. Click-toggle + outside-click-to-close is the standard accessible pattern. `useRef` + mousedown listener handles outside clicks without any dependency.
- **Test progress uses estimated pace (8s/case)** — The test run API is a single POST with no server-sent events. Real progress would require SSE or polling. Estimated-pace is the right tradeoff: it gives honest "approximately where are we" feedback without misleading precision or infrastructure changes. Caps at 95% to avoid a stuck-at-100% appearance before the API responds.
- **Progress bar replaces button while running** — Disabling the button with a spinner leaves the UI in a dead state for 30–90s. Replacing it with a contextual progress block gives architects confidence the run is proceeding and roughly how long to wait.
