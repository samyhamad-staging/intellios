## Action Log

### All-Conversation Intake v2

**Problem:** The three-phase intake architecture (Phase 1 structured form → Phase 2 conversation → Phase 3 review) required users to fill out a form before any conversation could begin. This is anti-AI-native — it front-loads cognitive burden and delays the value of the AI interview. The form also diverged from the intake spec's intent, which was for context collection to happen through conversation.

**Solution:** Eliminate the Phase 1 form entirely. Context collection now happens through a dedicated conversation mode powered by a new `submit_intake_context` tool.

**Changes:**

- `src/lib/intake/tools.ts` — added `submit_intake_context` tool: Claude calls this once the user has confirmed all 6 context fields (agentName, agentPurpose, enterpriseDomain, intendedUsers, riskTolerance, regulatoryContext) through conversation. Tool validates the payload with Zod and saves it via a PATCH to the session API.

- `src/lib/intake/system-prompt.ts` — added `CONTEXT_COLLECTION_PROMPT` mode: when `currentContext` is null, the system prompt instructs Claude to collect context conversationally (not via form), confirm fields with the user, then call `submit_intake_context`. Once context is saved, the full intake governance probing prompt loads.

- `src/app/api/intake/sessions/[id]/chat/route.ts` — `currentContext` is now mutable within the route handler. Added `onContextSubmit` callback: when `submit_intake_context` fires, the handler saves context to the DB and runs classification synchronously, then returns the risk tier in the tool response so Claude can acknowledge it inline.

- `src/app/intake/[sessionId]/page.tsx` — removed the Phase 1 form branch entirely. New sessions go directly to conversation. 3 significant additions:
  1. **Cold start fix:** `INTAKE_OPENER` message pre-populated as the first assistant message on fresh sessions — eliminates the blank loading state users saw on session entry.
  2. **Phase stepper:** 3-step header indicator (Context · Requirements · Review) shows current phase visually.
  3. **Orientation panel:** the progress rail shows a 2-column orientation view (6 context areas + 7 blueprint sections) when context has not yet been set; transitions to the existing Blueprint Progress view once `submit_intake_context` fires.

- `src/components/intake/intake-progress.tsx` — orientation panel component added; readiness score hidden until score > 0 (prevents false 15% on fresh sessions with no context).

- `src/components/chat/chat-container.tsx` — stale empty-state message removed (was surfacing "Start a conversation" on sessions that already had messages due to a timing issue).

### Font Tokenization — Design System v1.2.1

**Problem:** The codebase had 90+ instances of `text-[10px]`, `text-[11px]`, and `text-[9px]` — arbitrary Tailwind size overrides that bypassed the design system type scale. This made it impossible to change small text sizes globally and created inconsistency across components.

**Solution:** Define two new design tokens in `globals.css` and systematically replace all arbitrary sizes.

- `src/app/globals.css` — added `--text-2xs: 10px` and `--text-xs-tight: 11px` to the `@theme` block; registered as `text-2xs` and `text-xs-tight` Tailwind utilities.
- 16 component files updated in batch 4 (nav, intake, blueprint, registry, help, chat).
- 32 page-level files updated to complete the pass.

Total: ~90+ replacements across 48 files. No behavioral change — purely typographic consistency.

## Decisions

- **All-conversation intake is the right long-term direction.** Form-then-chat is not AI-native. Removing the form makes Intellios feel like an AI product from first contact. The `submit_intake_context` tool pattern (Claude calls it after conversational confirmation) is cleaner than form submission + page transition.

- **Phase 1 form is not preserved as a fallback.** The all-conversation path handles all cases. No toggle or feature flag needed. Old sessions created with Phase 1 context still work — the system uses `currentContext` from the DB regardless of how it was originally set.

- **Font tokenization is complete.** All arbitrary small text sizes are now tokens. This closes the last gap in Design System v1.2 (color in session 070, badge in session 071, typography in this session).
