## Path A vs. Path B decision

The runbook's Phase 1 documents both paths for `CHROMIUM_REMOTE_EXECUTABLE_URL`. Path A (self-hosted on `intellios-artifacts-prod` S3 bucket) is the production-durable path; Path B (Sparticuz GitHub release URL) is the smoke-only fallback.

### AWS preflight

```
$ aws sts get-caller-identity
{
    "UserId": "AIDAX3X7SM3SYBS5N5VTV",
    "Account": "540628117221",
    "Arn": "arn:aws:iam::540628117221:user/automation/intellios-automation-user"
}
exit 0  ✅

$ aws s3api head-bucket --bucket intellios-artifacts-prod
HTTP 404 — Not Found
exit 254  ❌

$ aws s3api get-bucket-location --bucket intellios-artifacts-prod
NoSuchBucket: The specified bucket does not exist
exit 254  ❌
```

**IAM principal:** `intellios-automation-user` — same as Session 170's `iam:PutUserPolicy` denial precedent. Identity itself is fine.

**Bucket finding:** `intellios-artifacts-prod` does not exist in account `540628117221`. The bucket is referenced aspirationally in `docs/runbooks/backup-restore.md` and the deployment guide, but never provisioned. The S3 helper at `src/lib/storage/s3.ts:15` reads the bucket name purely from `process.env.ARTIFACT_BUCKET`; `src/lib/env.ts:31` marks it `optional()`. With `ARTIFACT_BUCKET` unset in Vercel, **all evidence-package cache writes have been gracefully no-op'ing in production since the JSON export shipped** — every export rendering fresh bytes. Documented operational fact, not a bug.

### Decision: Path B for the smoke

Bucket provisioning is operationally sound but architecturally adjacent. Region, Object Ownership posture, versioning, lifecycle policy, and the evidence-cache co-location question all deserve their own ADR — provisioning an S3 bucket as a side-effect of a smoke test inverts the order ADR-029 was written to enforce. Path B unblocks Session 174 today; Path A migration becomes a future session with proper scoping.

`CHROMIUM_REMOTE_EXECUTABLE_URL` for the smoke:

```
https://github.com/Sparticuz/chromium/releases/download/v131.0.1/chromium-v131.0.1-pack.x64.tar
```

Bonus: with `ARTIFACT_BUCKET` unset in Vercel, the PDF route's S3 cache is also a no-op, which means every smoke render is fresh bytes. Preferable for the Phase 4 visual-fidelity comparison — no cached first-render polluting a second look.

Filed [**SCRUM-47**](https://samyhamad.atlassian.net/browse/SCRUM-47) — _"Provision intellios-artifacts-prod and migrate CHROMIUM_REMOTE_EXECUTABLE_URL from Path B to Path A"_ — pre-smoke so the link is present in this log.
