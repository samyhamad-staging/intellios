### W3-08: Accessibility Hardening

**Investigation:**

- Read `src/components/nav/mobile-layout.tsx` ‚Äî `<main>` exists but has no `id`, no skip target, no `aria-label`. Mobile hamburger button is `p-1.5` (~32px touch target, below 44px WCAG minimum).
- Read `src/components/ui/table.tsx` ‚Äî `TableHeader` renders `<th>` with no `scope` attribute. Added `scope="col"` as default prop.
- Read `src/components/ui/skeleton.tsx` ‚Äî `SkeletonList` renders a plain `<div>` with no ARIA attributes; screen readers have no signal that content is loading.
- Confirmed `InlineAlert` already has `role="alert"` (W3-04). 0 `aria-live` regions exist in the codebase.
- Confirmed `wflowsLoaded` state references in `registry/page.tsx` were stale after W3-07 React Query migration (the variable was removed from state but two JSX conditionals still referenced it).

**Changes:**

**`src/components/nav/mobile-layout.tsx`:**
1. Skip link: `<a href="#main-content" className="sr-only focus:not-sr-only focus:fixed ...">Skip to main content</a>` added before the layout `<div>`. Uses Tailwind's `sr-only` / `focus:not-sr-only` pattern ‚Äî invisible until focused by keyboard, then appears as an indigo pill in the top-left corner.
2. `<main>` gains `id="main-content"` (skip target) and `aria-label="Main content"`.
3. Mobile hamburger button: `p-1.5` ‚Üí `p-2.5` + `min-w-[44px] min-h-[44px] flex items-center justify-center` ‚Äî meets WCAG 2.1 AA 44√ó44px minimum touch target.
4. `aria-expanded={sidebarOpen}` and `aria-controls="mobile-sidebar"` added to hamburger button.
5. Drawer `<div>` gains `id="mobile-sidebar"` (referenced by `aria-controls`).

**`src/components/ui/table.tsx`:**
6. `TableHeader` prop signature updated: `scope?: "col" | "row" | "colgroup" | "rowgroup"` defaulting to `"col"`. `<th scope={scope} ...>` ‚Äî all table headers now have correct scope by default. Callers can override for row-header scenarios.

**`src/components/ui/skeleton.tsx`:**
7. `SkeletonList` gains `role="status"`, `aria-label="Loading content"`, `aria-live="polite"`, `aria-busy="true"`, and a `<span className="sr-only">Loading‚Ä¶</span>` child so screen readers announce the loading state.

**`src/components/ui/visually-hidden.tsx` (new):**
8. `VisuallyHidden` ‚Äî renders children with `className="sr-only"`. Accepts `as` prop for semantic tag choice. Use for icon-only button labels or supplementary screen-reader text.
9. `LiveRegion` ‚Äî renders a polite `role="status"` / `aria-live="polite"` / `aria-atomic="true"` `<div className="sr-only">`. Accepts `message` prop; updating `message` triggers a screen reader announcement without moving focus.

**`src/app/registry/page.tsx`:**
10. Fixed stale `wflowsLoaded` references (two JSX conditionals referenced a state variable removed in the React Query migration).
11. Added `<div aria-live="polite" aria-atomic="false">` wrapper around loading/error/content region.
12. Error div gains `role="alert"`.

**`src/app/governance/page.tsx`:**
13. Policy Library loading skeleton gains `role="status"`, `aria-label="Loading policies"`, `aria-busy="true"`, `<span className="sr-only">Loading policies‚Ä¶</span>`.
14. Added `<div aria-live="polite" aria-atomic="false">` wrapper around Policy Library loading/content region.
