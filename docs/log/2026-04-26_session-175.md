The 3.4× size delta likely reflects different font-embedding behavior between Chromium's print path and whatever produced the reference (`samples/build_evidence_pdf.py` is colocated — looks like a Python script that produced the reference, not an Intellios-era renderer). Worth noting; not a fidelity flag.

**Correction:** the form-feed-counting heuristic in this session's structural-check capture initially reported 14 pages (`grep -c $'\f' /tmp/pdf_text.txt` returned 13, plus 1 for the implicit final page = 14). `pdfinfo` and the rendered footer "Page N of 13" agree on 13. The +1 in the heuristic was off — `pdftotext`'s output emits one form-feed *between* pages, so an N-page PDF has N−1 form-feeds (for the gaps) but the heuristic should have been `count_of_FF + 1` only if there's content after the last FF, which there is by default — so 13 form-feeds + 1 final-page = 14. Wrong. The actual page count is `count_of_FF` (since `pdftotext` also emits a trailing FF after the last page in some toolchain versions), which here was 13 — but heuristics like this are fragile across pdftotext versions. The right fix is `pdfinfo`, which we'll have once [SCRUM-49](https://samyhamad.atlassian.net/browse/SCRUM-49) lands. Per-page-first-line capture below is unaffected (drove off the same form-feed split).

## Surprises / data captured along the way

1. **`ed34ef1a` is in local DB.** Lifted there during the Session 170 walkthrough; currently `deprecated`. Used as the local smoke target — same blueprint name as the runbook, just a different status. The renderer doesn't care; status gate is route-layer concern.
2. **Deterministic-seed blueprints (id `2000…`) have malformed ABP `metadata.id` UUIDs.** They fail `readABP`'s Zod UUID validator → `assembleMRMReport` throws. Skipped via the candidate-walk. Reproducible, not a renderer bug. Filed below.
3. **`scripts/seed-demo.ts` env loader has a soft bug.** Its `if (!(key in process.env))` guard skips overwriting when the key exists with an empty string (a common shell-inheritance state for `ANTHROPIC_API_KEY`). Surfaced when seed-demo failed env-validation despite a populated `.env.local`. Workaround used: `set -a; source .env.local; set +a` before running. Filed below.
4. **`tsx` does not auto-load `.env.local`.** `@/lib/env` validates DB / Auth / Anthropic envs at import time via Zod. Solutions: source first (used here), or add a self-loading prelude to scripts. The renderer smoke harness inherits the same constraint — documented in its docstring.
5. **Render time 12.5 s is cold-start.** Playwright bundled Chromium (`chromium-1217`) had to be installed (`npx playwright install chromium`) — 130 MB, one-time. Actual launch cost on a populated cache is much lower; subsequent local renders should be 2–4 s.
6. **No empty-string fallthrough on `ARTIFACT_BUCKET` locally either.** Mirrors production. Cache no-op. Not a regression.
