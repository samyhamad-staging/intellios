## Summary

**Deployment Confirmation Modal** — Every production deployment now requires a change reference number, optional deployment notes, and an explicit authorization acknowledgment. All fields flow through to the existing audit log as metadata on the `blueprint.status_changed` event. No route changes beyond the Zod schema extension.

**Global Search** — Both the Agent Registry and Pipeline Board now support real-time text search by agent name, agentId (partial), or tag. The Registry also adds a status filter dropdown. Filtering is entirely client-side (`useMemo`) over already-loaded data.

**Review Decision Banner** — The Blueprint detail page now prominently displays the last review decision as a color-coded banner (green for approved, red for rejected, amber for changes requested), including the reviewer's identity, timestamp, and comment. Visible immediately on page load without navigating to the audit trail or a separate tab.

**SOD Role Guard on Deployed Transition** — A control validation pass identified that the `deployed` transition had no role restriction, allowing a designer to self-promote their own approved blueprint to production by calling the API directly with a `changeRef`. Fixed by adding a `reviewer/admin`-only guard alongside the existing `in_review` and `deprecated` guards. The deployment path now has three independent enforcement layers: client redirect → `changeRef` gate → RBAC role check.
