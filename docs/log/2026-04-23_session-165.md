## Work Stream B — ADR-031

ADR-031 authored at `docs/decisions/031-sandbox-vs-customer-iam-posture.md`.

**Decision:** Two explicit postures, not one. Sandbox = `AmazonBedrockFullAccess` (ergonomics for solo-founder bootstrapping). Customer-account = scoped inline policy via `put-role-policy` (minimum-necessary, fails closed if `iam:PutRolePolicy` denied). Provisioner script must support both via `--scoped` flag.

Supporting artifacts created:
- `docs/infra/iam-policies/bedrock-exec-role-scoped.json` — scoped inline policy (lifted from session 164 log)
- `docs/infra/iam-policies/bedrock-exec-role-trust.json` — trust policy
- `docs/infra/iam-policies/README.md` — when to use which posture
